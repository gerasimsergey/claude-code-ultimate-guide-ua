**Критична деталь встановлення — співіснування хуків**:

`claude-mem` додає хуки на `SessionStart`, `PostToolUse`, `Stop` та `SessionEnd`. Якщо у вас уже є хуки в `settings.json`, **claude-mem не об'єднає їх автоматично** — він перезапише масиви хуків.

Перед встановленням:
1. Зробіть резервну копію поточного `settings.json`
2. Занотуйте всі існуючі хуки (масиви PostToolUse, UserPromptSubmit)
3. Після встановлення вручну перевірте, чи містять масиви хуків І ваші існуючі хуки, І нові хуки claude-mem

```json
// ✅ Правильно — обидва хуки співіснують
"hooks": {
  "PostToolUse": [
    {"matcher": "...", "hooks": [{"type": "command", "command": "your-existing-hook.sh"}]},
    {"matcher": "...", "hooks": [{"type": "command", "command": "claude-mem-hook.sh"}]}
  ]
}

// ❌ Неправильно — claude-mem мовчки замінив ваші хуки
"hooks": {
  "PostToolUse": [
    {"matcher": "...", "hooks": [{"type": "command", "command": "claude-mem-hook.sh"}]}
  ]
}
```

**Надійність: архітектура fail-open (v9.1.0+)**:

Якщо процес воркера `claude-mem` впав (краш, перезапуск, конфлікт портів), Claude Code продовжує працювати нормально — він не блокується і не видає помилок. Сесії просто не фіксуються, поки воркер не перезапуститься.

```bash

# Перезапуск воркера вручну за потреби

```

### Graphify (Графи знань кодобази)

**GitHub**: [safishamsi/graphify](https://github.com/safishamsi/graphify) | **PyPI**: `graphifyy` | **Зірок**: 42K | **Ліцензія**: MIT

Graphify перетворює директорію проекту на персистентний граф знань та впорскує компактний структурний звіт (`GRAPH_REPORT.md`) у Claude Code. Асистент відповідає на питання про архітектуру, читаючи заздалегідь побудований граф, замість повторного сканування файлів. Початкове вилучення виконується один раз; наступні сесії запитують граф із майже нульовими витратами токенів.

**Як це працює**: Три проходи за один запуск:

1. **Локальне вилучення AST** — tree-sitter парсить 25+ мов (Python, TS, Go, Rust, Java тощо) у граф викликів. Без витрат на API, без мережі.
2. **Опціональна локальна транскрипція** — faster-whisper транскрибує аудіо/відео локально.
3. **Паралельне семантичне вилучення** — субагенти Claude обробляють доки, PDF та зображення за вашим ключем API.

Результати об'єднуються в граф NetworkX, кластеризований за алгоритмом Лейдена (на основі топології — без векторних ембеддінгів). Кожен зв'язок позначається як `EXTRACTED` (вилучено), `INFERRED` (виведено) або `AMBIGUOUS` (неоднозначно). Три вихідні файли потрапляють у `graphify-out/`:

- `graph.html` — інтерактивна візуалізація в браузері.
- `GRAPH_REPORT.md` — вузли-концентратори, міжфайлові зв'язки, пропоновані запити (впорскується в Claude).
- `graph.json` — персистентний файл, доступний для запитів без повторного вилучення.

**Встановлення**:

```bash
# Потребує Python 3.10+. Пакет PyPI: graphifyy (дві 'y'). CLI: graphify.
uv tool install graphifyy && graphify install

# Постійна інтеграція з Claude Code (записує розділ CLAUDE.md + хук)
graphify claude install

# Побудова графа для поточної директорії
/graphify .

# Інкрементальне оновлення (SHA256 хешування)
/graphify . --update

# Прямі запити до графа
/graphify query "що з'єднує auth із шаром бази даних?"
/graphify path "UserService" "DatabasePool"
/graphify explain "RateLimiter"

# Автоматична перебудова при комітах (тільки локальний AST, 0 витрат API)
graphify hook install
```

**Опціональні доповнення**:
```bash
pip install "graphifyy[office]"   # підтримка .docx, .xlsx
pip install "graphifyy[video]"    # транскрипція .mp4, .mov, .mp3
```

**Graphify vs GrepAI**: Різні рівні, що доповнюють один одного. GrepAI (Ollama, локально, безкоштовно) обробляє семантичні пошуки в реальному часі — швидко, цілеспрямовано. Graphify попередньо обчислює структурні зв'язки (ланцюжки викликів, міжфайлові залежності) та замінює повторне читання файлів одним звітом на сесію. GrepAI для відкриттів; Graphify для архітектурних міркувань.

**Graphify vs claude-mem**: Різні завдання. `claude-mem` зберігає те, що ви *обговорювали* (рішення, спостереження). Graphify мапує те, що *містить кодобаза* (структура, залежності). Вони не перетинаються.

**Воркфлоу команди**: Комітьте `graphify-out/` (крім `manifest.json` та `cache/`) у git, щоб колеги успадкували готовий граф без запуску вилучення.

**Застереження**:
- Не-кодові файли (PDF, зображення) надсилаються на API при першому запуску — витрати накопичуються на великих репозиторіях з медіа.
- Без пост-коміт хука граф втрачає актуальність при зміні коду.
- Заяви автора про ефективність токенів (71.5x–120x менше) є самозвітами без незалежної перевірки. Сприймайте як орієнтир.

### mgrep (Альтернативний семантичний пошук)

**Призначення**: Семантичний пошук природною мовою по коду, PDF та зображеннях.

**Чому розглянути mgrep**: Якщо потрібен **мультиформатний пошук** (код + PDF + зображення) або ви віддаєте перевагу хмарному рішенню. Бенчмарки показують ~2x менше витрачених токенів порівняно з grep-воркфлоу.

**Ключові функції**:
- Семантичний пошук за описом.
- Фонова індексація `mgrep watch` з урахуванням `.gitignore`.
- Мультиформатність (код, PDF, зображення, текст).

**Коли використовувати**:
- Пошук у змішаному контенті.
- Перевага хмарним ембеддінгам над локальною Ollama.

### ast-grep (Структурний пошук коду)

**Призначення**: Пошук за патернами AST для точних структурних запитів.

**Встановлення**:
```bash
npx skills add ast-grep/agent-skill
# Або через маркетплейс плагінів: /plugin marketplace add
```

**Що це?**
ast-grep шукає код на основі **синтаксичної структури** (Abstract Syntax Tree), а не простого тексту. Це дозволяє знаходити патерни типу "async-функції без try/catch" або "React-компоненти з певними хуками", які regex не може надійно розпізнати.

| Аспект | Поведінка |
|--------|----------|
| **Виклик** | **Явний** - Claude не знає автоматично, коли його використовувати |
| **Інтеграція** | Плагін, що вчить Claude писати правила ast-grep |
| **Мови** | JS, Python, Rust, Go, Java, C++, Ruby, PHP тощо |
| **Збіг патернів** | Метазмінні (`$VAR`), реляційні запити, композитна логіка |

**Коли використовувати**:
- ✅ Масштабний рефакторинг (>50k рядків).
- ✅ Міграції фреймворків.

### agent-browser (Vercel Labs) — ШІ-нативна автоматизація

**Призначення**: Headless-браузер, створений для ШІ-агентів. Оптимізує весь вивід для споживання LLM. Написаний на Rust.

**Чому це важливо**: Playwright MCP багатослівний — кожен знімок DOM додає токени. `agent-browser` повертає лише активні елементи через стабільні короткі посилання (`@e1`, `@e2`), скорочуючи витрати токенів на **~82.5%**.

**Встановлення**: `brew install vercel-labs/tap/agent-browser` або `npm install -g @vercel-labs/agent-browser`.

**Патерн "Ralph Wiggum Loop"** (агент, що сам себе перевіряє):
1. Агент пише код фічі.
2. Деплоїть (напр. на Vercel).
3. `agent-browser` автономно переходить за URL.
4. Тестує сценарії, читає знімки доступності.
5. При помилці: агент читає вивід, виправляє код, передеплоює.
6. Цикл до повного успіху — без участі людини.

### doobidoo Memory Service (Семантична пам'ять)

**Призначення**: Персистентна семантична пам'ять із пошуком між сесіями та підтримкою багатьох клієнтів.

**Чому доповнює Serena**:
- Serena: Ключ-значення (треба знати ключ).
- doobidoo: Семантичний пошук ("що ми вирішили про auth?") — знаходить за змістом.

| Функція | Serena | doobidoo |
|---------|--------|----------|
| Зберігання | Ключ-значення | Семантичні ембеддінги |
| Пошук за змістом | Ні | Так |
| Мульти-клієнт | Тільки Claude | 13+ додатків |

**Навіщо**: Рішення між сесіями, "що ми вирішили про X?", спільна пам'ять для Claude, Cursor та VS Code.

### Kairn: Пам'ять-граф із біологічним розпадом

**Призначення**: Довгострокова пам'ять проекту у вигляді графа знань з автоматичним застаріванням — неактуальна інформація зникає сама, не засмічуючи контекст.

**Відмінності**:
- **Типізовані зв'язки**: `depends-on`, `resolves`, `causes` — фіксує причинність.
- **Біологічний розпад**: рішення живуть ~200 днів, "милиці" (workarounds) ~50 днів — автоочищення без ручного видалення.
- **18 інструментів MCP**: операції з графом, управління досвідом.

### ICM: Подвійна архітектура пам'яті (Rust, 0 залежностей)

**Призначення**: Персистентна пам'ять для ШІ-агентів, що поєднує епізодичний розпад (Memories) та постійний граф знань (Memoirs) в одному швидкому Rust-бінарі.

**Чому ICM**:
- Жодних проблем із Python-залежностями.
- Встановлення через Homebrew без налаштування оточення.
- Поєднання епізодичної та постійної пам'яті.
- Підтримка 14 клієнтів (Claude Code, Cursor, VS Code, Zed тощо).
