**Використовуйте Dual-Instance, коли**:
- Потрібна верифікація плану перед виконанням.
- Специфікації складні або розмиті (допомагає уточнення через інтерв'ю).
- Бюджет нижчий за патерн Бориса ($100-200/міс).
- Якість важливіша за швидкість.

#### Архітектура Dual-Instance: Два інстанси, дві ролі

```
┌─────────────────────────────────────────────────────┐
│         АРХІТЕКТУРА DUAL-INSTANCE                   │
├─────────────────────────────────────────────────────┤
│  ┌──────────────────┐                               │
│  │  Claude Zero     │  Планування та Рев'ю          │
│  │  (Planner)       │  - Досліджує кодобазу         │
│  └────────┬─────────┘  - Пише плани                 │
│           │            - НІКОЛИ не торкається коду  │
│           ▼                                          │
│  ┌──────────────────┐                               │
│  │  Claude One      │  Реалізація                   │
│  │  (Implementer)   │  - Читає схвалені плани       │
│  └──────────────────┘  - Пише код та робить коміти  │
└─────────────────────────────────────────────────────┘
```

**Етапи воркфлоу**:
1. **Планування (Claude Zero)**: Використовує `/plan`, задає уточнюючі питання, створює "agent-ready" план у `.claude/plans/Review/`.
2. **Перевірка людиною**: Ви рецензуєте план. Якщо все ок — переносите файл в папку `Active/`.
3. **Реалізація (Claude One)**: Читає план, виконує кроки, звітує про завершення.
4. **Верифікація (Claude Zero)**: Перевіряє, чи відповідає код плану та чи пройдені тести.
5. **Архів**: Перенесення плану в `Completed/`.

**Аналіз витрат**: Для складних фіч (де зазвичай потрібно ≥2 ітерації виправлень) Dual-Instance дешевше, бо запобігає накопиченню помилок та галюцинацій завдяки чітким інструкціям у плані.

### Фундамент: Git Worktrees (Обов'язково)

Мультиагентні воркфлоу **ВИМАГАЮТЬ** використання git worktrees. Без них паралельні інстанси створять "пекло мерджу".
**Переваги**:
- Кожен інстанс працює в ізольованому стані Git.
- Немає втрати контексту при перемиканні гілок.
- Миттєве створення середовища (~1с).

#### Просунуті інструменти: Worktrunk
Це Rust CLI, що спрощує керування деревами:
- **Project hooks**: Автоматизація дій після створення дерева.
- **Статус CI**: Перегляд результатів білду прямо у списку дерев (`wt list`).
- **LLM коміти**: Генерація повідомлень через `wt commit`.

### Посібник з імплементації (Прогресивне масштабування)
- **Фаза 1**: Майстерність одного інстансу (успіх >80% без правок).
- **Фаза 2**: Тестування двох інстансів (Backend + Frontend ізольовано).
- **Фаза 3**: Мультиагентна система (3-5 інстансів) з фреймворком оркестрації.

### Моніторинг та Обсервабельність
Використовуйте HTTP-заголовок **`X-Claude-Code-Session-Id`** (v2.1.86+) для відстеження витрат та лімітів на рівні проксі (Nginx/Envoy) без втручання в конфігурацію Claude Code.

**Сигнали до відкату (Rollback)**:
- Конфлікти мерджу >5% від усіх PR.
- Перевантаження розробника мікроменеджментом агентів (>30% часу).
- Зниження якості тестів або поява "флакі"-тестів.
