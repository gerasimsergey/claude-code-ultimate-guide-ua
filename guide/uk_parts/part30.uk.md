**Приклад скрипта: capture-instincts.sh**
Цей скрипт запускається через хук `Stop` і витягує з логів сесії спостереження про те, що спрацювало найкраще, не створюючи затримок для користувача під час роботи.
Він додає "інстинкти" у файл `pending.yaml`, які ви потім можете переглянути та перетворити на постійні правила в `CLAUDE.md`.

### 9.25 Harness Engineering (Проектування каркаса)

**Головна ідея**: Надійність виконання залежить не від якості моделі, а від інфраструктури навколо неї. Ця інфраструктура називається **Harness (Каркас)**.
Каркас складається з 5 підсистем:
1. **Інструкції** (CLAUDE.md).
2. **Інструменти** (Bash, Edit).
3. **Середовище** (init.sh, devcontainers).
4. **Стан** (feature_list.json, progress.md).
5. **Зворотний зв'язок** (тести, лінтери).

**Правило WIP=1**: Одне завдання за раз. Агент має завершити одну фічу через усі шари перевірки, перш ніж братися за наступну. Це критично підвищує відсоток успішного завершення проектів.

**Життєвий цикл сесії**:
`READ → INIT → RESUME → SELECT → EXECUTE → VERIFY → UPDATE → LOG → CLEANUP → COMMIT`.

- `Ctrl+R`: Пошук в історії команд.

### 10.3 Довідник конфігурації CLI
Використовуйте прапорці при запуску `claude`:
- `-c`: Продовжити останню розмову.
- `-p "prompt"`: Виконати команду в неінтерактивному режимі (для скриптів).
- `--model opus`: Змінити модель.
- `--dangerously-skip-permissions`: Пропускати всі запити на дозвіл (лише для CI/CD!).
- `--worktree`: Запустити в ізольованій гілці Git.

### 10.4 Усунення несправностей (Troubleshooting)

**Швидка діагностика**:
1. Перевірте контекст: `/status`. Якщо >70% — `/compact`.
2. Перевірте стан системи: `/doctor`.
3. Помилки MCP: `claude mcp list`.

**Поширені помилки MCP**:
- **Invalid characters**: Імена серверів мають містити лише літери, цифри та дефіси.
- **Windows Paths**: Використовуйте прямі слеші `/` або екрануйте зворотні `\\` у конфігах.
- **Standalone Test**: Спробуйте запустити сервер через `npx` напряму в терміналі, щоб побачити логи помилок.
