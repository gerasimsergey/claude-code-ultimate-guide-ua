# Контекстна інженерія

Як заповнити вікно контексту Claude правильною інформацією в потрібний час.

---

### 3-рівнева система контексту

Контекстна інженерія працює на 3 окремих рівнях. Розуміння того, який рівень обрати, запобігає перевантаженню контексту.

```mermaid
flowchart TD
    subgraph GLOBAL["🌍 Рівень 1: Глобальний — ~/.claude/CLAUDE.md"]
        G1["Айдентика та преференції тону"]
        G2["Універсальні налаштування інструментів"]
        G3["Конвенції коду для всіх проектів"]
        G4["Ціль: до 200 рядків"]
    end

    subgraph PROJECT["📁 Рівень 2: Проектний — ./CLAUDE.md + модулі"]
        P1["Стек та архітектурні рішення"]
        P2["Командні конвенції + правила деплою"]
        P3["@imports для окремих підсистем"]
        P4["Ціль: до 150 рядків (корінь) + модулі"]
    end

    subgraph SESSION["⚡ Рівень 3: Сесія — інструкції в чаті, /add"]
        S1["Обмеження для конкретного завдання"]
        S2["Тимчасові перезаписи правил"]
        S3["Ефемерно — не зберігається після сесії"]
    end

    GLOBAL --> PROJECT --> SESSION

    OVR["Пріоритет: Сесія > Проект > Глобально<br/>Більш специфічне перемагає загальне"] -.-> SESSION

    style G1 fill:#E87E2F,color:#fff
    style P1 fill:#6DB3F2,color:#fff
    style S1 fill:#F5E6D3,color:#333
    style OVR fill:#7BC47F,color:#333

    click G1 href "../core/context-engineering.uk.md#3-ієрархія-конфігурації" "Глобальний рівень"
```

### Бюджет контексту та деградація дотримання правил

Дотримання правил `CLAUDE.md` погіршується зі зростанням об'єму файлу. Після ~150 правил моделі починають вибірково ігнорувати інструкції.

```mermaid
flowchart LR
    subgraph ZONES["Дотримання правил за кількістю рядків"]
        Z1["1–100 рядків<br/>~95% дотримання ✓"]
        Z2["100–200 рядків<br/>~88% дотримання"]
        Z3["200–400 рядків<br/>~75% дотримання ⚠️"]
        Z4["400–600 рядків<br/>~60% дотримання"]
        Z5["600+ рядків<br/>~45% і падає ✗"]
    end

    Z1 --> Z2 --> Z3 --> Z4 --> Z5

    FIX["✅ Рішення: Path-scoping<br/>Корінь: тільки спільні правила<br/>Підсистеми: модулі за потребою<br/>Результат: зменшення контексту на 40-50%<br/>Дотримання правил у зеленій зоні"] -.-> Z1

    style Z1 fill:#7BC47F,color:#333
    style Z3 fill:#E87E2F,color:#fff
    style Z5 fill:#E85D5D,color:#fff
    style FIX fill:#7BC47F,color:#333

    click Z1 href "../core/context-engineering.uk.md#2-бюджет-контексту" "Бюджет контексту"
```

### Монолітна vs Модульна архітектура

Монолітний `CLAUDE.md` — найпоширеніша помилка. Модулі з прив'язкою до шляхів (path-scoping) завантажують тільки те, що актуально для завдання.

```mermaid
flowchart TD
    subgraph BAD["❌ Анти-патерн: Монолітний CLAUDE.md"]
        B1(["CLAUDE.md (600 рядків)<br/>API + БД + UI + Деплой<br/>— все впереміш"])
        B2("Всі 600 рядків завантажуються<br/>для кожного файлу")
        B3("Правила 500+ отримують<br/>лише ~30% уваги")
        B4(["Дотримання правил деградує"])
        B1 --> B2 --> B3 --> B4
        style B1 fill:#E85D5D,color:#fff
        style B4 fill:#E85D5D,color:#fff
    end

    subgraph GOOD["✅ Найкраща практика: Модулі"]
        G1(["Корінь CLAUDE.md (~100 р.)<br/>Спільні правила + @import"])
        G2["src/api/CLAUDE-api.md<br/>Тільки для API"]
        G3["src/components/CLAUDE-ui.md<br/>Тільки для компонентів"]
        G4(["Кожен модуль: повний фокус<br/>Контекст на 40–50% менший"])
        G1 --> G2
        G1 --> G3
        G2 & G3 --> G4
        style G1 fill:#7BC47F,color:#333
        style G4 fill:#7BC47F,color:#333
    end

    click B1 href "../core/context-engineering.uk.md#4-модульна-архітектура" "Модульна архітектура"
```

### Дерево рішень для розміщення правил

Кожне нове правило має опинитися на правильному рівні.

```mermaid
flowchart TD
    A([Нове правило для розміщення]) --> B{Актуально для\nкожного вашого\nпроекту?}
    B -->|Так| C([Глобальний CLAUDE.md<br/>~/.claude/CLAUDE.md])

    B -->|Ні| D{Стосується\nокремих файлів\nчи підсистем?}
    D -->|Так| E([Модуль підсистеми<br/>напр. src/api/CLAUDE-api.md])

    D -->|Ні| F{Це процедурний\nпокроковий\nворкфлоу?}
    F -->|Так| G([Файл скіла (Skill)<br/>.claude/skills/task.md<br/>Завантажується за запитом])

    F -->|Ні: стандарт/правило| H{Стосується\nвсього проекту?}
    H -->|Так| I([Корінь CLAUDE.md проекту<br/>./CLAUDE.md])
    H -->|Ні: на один раз| J([Інструкція в сесії<br/>Скажіть Claude прямо зараз])

    RULE["Золоте правило:<br/>Якщо пишете це вдруге —<br/>час перенести в постійний рівень"] -.-> J

    style B fill:#E87E2F,color:#fff
    style C fill:#E87E2F,color:#fff
    style E fill:#6DB3F2,color:#fff
    style J fill:#F5E6D3,color:#333
    style RULE fill:#7BC47F,color:#333

    click A href "../core/context-engineering.uk.md#3-ієрархія-конфігурації" "Розміщення правил"
```
