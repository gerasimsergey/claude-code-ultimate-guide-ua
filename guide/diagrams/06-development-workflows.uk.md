### TDD (Red-Green-Refactor) з Claude

Розробка через тестування (TDD), адаптована для Claude Code: спочатку пишемо тест, що не проходить, потім просимо Claude реалізувати мінімум коду для його успішного виконання.

```mermaid
flowchart TD
    A([Початок: Нова функція]) --> B(Написання тесту, що не<br/>проходить (людина))
    B --> C(Запуск тестів)
    C --> D{Тести не проходять<br/>як очікувалося?}
    D -->|Ні: проходять!| E(Виправте тест — він заслабкий)
    E --> B
    D -->|Так: RED ✓| F(Просимо Claude реалізувати<br/>мінімальний код)
    F --> G(Запуск тестів знову)
    G --> H{Тести проходять?}
    H -->|Ні| I(Діагностика з Claude<br/>виправлення реалізації)
    I --> G
    H -->|Так: GREEN ✓| J{Код потребує<br/>рефакторингу?}
    J -->|Так| K(Рефакторинг з Claude)
    K --> L(Запуск тестів: все ще зелені?)
    L -->|Ні| I
    L -->|Так: REFACTOR ✓| M{Потрібні ще<br/>функції?}
    J -->|No| M
    M -->|Так| B
    M -->|Ні| N([Функція готова ✓])

    style B fill:#E85D5D,color:#fff
    style F fill:#E85D5D,color:#fff
    style G fill:#7BC47F,color:#333
    style N fill:#7BC47F,color:#333

    click A href "../workflows/tdd-with-claude.uk.md" "TDD — Нова функція"
```

### Пайплайн розробки Spec-First

Специфікація пишеться перед кодом. Claude використовує її як єдине джерело істини.

```mermaid
flowchart LR
    A([Ідея / Вимога]) --> B(Написання spec.md<br/>природною мовою)
    B --> C(Claude перевіряє специф.<br/>на ясність + повноту)
    C --> D{Специф. схвалена<br/> людиною?}
    D -->|Ні: є прогалини| E(Уточнення специф.<br/>заповнення прогалин)
    E --> C
    D -->|Так| F(Генерація тестів<br/>на основі специф.)
    F --> G(Генерація реалізації<br/>з специф. + тестів)
    G --> H(Запуск тестів)
    H --> I{Усі тести<br/>проходять?}
    I -->|Ні| J(Claude виправляє<br/>реалізацію)
    J --> H
    I -->|Так| K(Перегляд людиною<br/>специф. vs результат)
    K --> L{Відповідає<br/>специф.?}
    L -->|Ні| M(Оновлення специф.<br/>або реалізації)
    M --> K
    L -->|Так| N([Мердж ✓])

    style A fill:#F5E6D3,color:#333
    style B fill:#6DB3F2,color:#fff
    style N fill:#7BC47F,color:#333

    click B href "../workflows/spec-first.uk.md" "Написання spec.md"
```

```
                              │
                        Схвалено? ─Ні→ Уточнення
                              │ Так

                              │

                              │

                              │ Так
                        Перегляд → Ок? ─Ні→ Виправлення
                              │ Так
                            Мердж ✓
```

### Воркфлоу на основі планів (Plan-Driven)

Для складних завдань: Claude вивчає код, пропонує план, ви його коментуєте, а потім Claude виконує тільки схвалені кроки.

```mermaid
flowchart TD
    A([Складне завдання]) --> B(Вхід у режим Plan<br/>Shift+Tab × 2)
    B --> C(Claude вивчає<br/>структуру кодобази)
    C --> D(Claude пропонує план<br/>із списком файлів)
    D --> E(Людина перевіряє план)
    E --> F{План<br/>прийнятний?}
    F -->|Ні| G(Людина коментує план<br/>вносить правки)
    G --> H(Claude оновлює план)
    H --> E
    F -->|Так| I(Схвалення плану<br/>Вихід з режиму Plan)
    I --> J(Claude виконує<br/>крок за кроком)
    J --> K(Claude звітує<br/>про прогрес)
    K --> L{Неочікувана<br/>проблема?}
    L -->|Так| M(Claude сигналізує<br/>просить допомоги)
    M --> F
    L -->|Ні| N{Усі кроки<br/>виконано?}
    N -->|Ні| J
    N -->|Yes| O([Готово ✓])

    style B fill:#6DB3F2,color:#fff
    style O fill:#7BC47F,color:#333

    click B href "../ultimate-guide.uk.md#23-режим-планування-plan-mode" "Режим Plan"
```

### Цикл ітераційного вдосконалення

Результат рідко буває ідеальним з першої спроби. Цей цикл допомагає систематично покращувати результати через точний фідбек.

```mermaid
flowchart TD
    A([Перший промпт]) --> B(Claude генерує результат)
    B --> C(Оцінка якості)
    C --> D{Достатньо<br/>добре?}
    D -->|Так| E([Готово ✓])
    D -->|Ні| F(Пошук конкретної проблеми<br/>Що саме не так?)
    F --> G{Тип проблеми?}
    G -->|Стиль/тон| H(Додати: обмеження стилю)
    G -->|Бракує інфо| I(Додати: відсутній контекст)
    G -->|Хибний підхід| J(Додати: зміна підходу)
    G -->|Обсяг| K(Додати: обмеження довжини)
    H --> L(Уточнення інструкції)
    I --> L
    J --> L
    K --> L
    L --> M(Claude оновлює результат)
    M --> N(Порівняння До/Після)
    N --> O{Є покращення?}
    O -->|Так| C
    O -->|Ні| P(Потрібен інший<br/>підхід)
    P --> F

    style A fill:#F5E6D3,color:#333
    style E fill:#7BC47F,color:#333
    style L fill:#6DB3F2,color:#fff

    click A href "../workflows/iterative-refinement.uk.md" "Перший промпт"
```

### AI Fluency — Шляхи Високої та Низької Майстерності

Згідно з індексом AI Fluency (Anthropic, 2026), 70% користувачів потрапляють у "Парадокс Артефакту" — приймають перший результат без перевірки. Висока майстерність (30%) полягає в ітераціях та критичному аналізі.

```mermaid
flowchart TD
    A([Запит користувача до Claude]) --> B(Claude генерує результат<br/>код · файл · план)
    B --> C["⚠️ Парадокс Артефакту<br/>Привабливий вигляд присипляє<br/>критичне мислення"]

    C -->|"70% користувачів"| D(Прийняття результату<br/>без глибокої перевірки)
    C -->|"30% користувачів"| E(Ітерації + запитання<br/>визначення меж співпраці)

    D --> D1["Падіння активності:<br/>−5.2% пошук прогалин<br/>−3.7% перевірка фактів<br/>−3.1% критика логіки"]
    D1 --> D2([Приховані дефекти · пропущені вимоги])

    E --> E1("Критика результату:<br/>'Що ти пропустив?<br/>Які припущення зробив?'")
    E1 --> E2(Пошук прогалин<br/>Уточнення контексту)
    E2 --> E3{Задоволені?}
    E3 -->|Ні — знову| E1
    E3 -->|Так| E4([Надійний, перевірений результат ✓])

    E4 --> G["Ефект:<br/>в 5.6 разів більше виявлених помилок<br/>Джерело: Anthropic AI Fluency Index, 2026"]

    style C fill:#E85D5D,color:#fff
    style D fill:#E85D5D,color:#fff
    style E fill:#7BC47F,color:#333
    style E4 fill:#7BC47F,color:#333

    click B href "../ultimate-guide.uk.md#пастки-воркфлоу-та-парадокс-артефакту" "Claude генерує результат"
```
