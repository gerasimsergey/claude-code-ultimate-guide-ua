# Витрати та оптимізація

Як отримати максимум від Claude Code, контролюючи споживання токенів та витрати.

---

### Схема вибору моделі

Не всі завдання потребують найпотужнішої моделі. Вибір правильної моделі може зменшити витрати в 5-10 разів без втрати якості.

```mermaid
flowchart TD
    A([Завдання для виконання]) --> B{Складність?}

    B -->|Просте| C["Прості завдання:<br/>виправлення опечаток, перейменування,<br/>форматування, переклад"]
    C --> D([Haiku 4.5<br/>💰 Найдешевша, найшвидша<br/>~в 5 разів дешевша за Sonnet])

    B -->|Стандартне| E["Стандартні завдання:<br/>реалізація функцій,<br/>фікси багів, рефакторинг"]
    E --> F([Sonnet 4.5/4.6<br/>💰💰 Збалансована<br/>Краще співвідношення ціна/якість])

    B -->|Складне| G{Потрібне глибоке<br/>міркування?}
    G -->|Так| H["Складні завдання:<br/>архітектурні рішення,<br/>рев'ю безпеки,<br/>аналіз багатьох файлів"]
    H --> I([Opus 4.7 / 4.6<br/>💰💰💰 Найпотужніша<br/>~в 5 разів дорожча за Sonnet])

    G -->|Ні: просто об'ємне| J["Великі, але зрозумілі завд.:<br/>масштабний рефакторинг,<br/>генерація документації"]
    J --> F

    style A fill:#F5E6D3,color:#333
    style D fill:#7BC47F,color:#333
    style F fill:#6DB3F2,color:#fff
    style I fill:#E87E2F,color:#fff

    click A href "../ultimate-guide.uk.md#25-вибір-моделі-та-посібник-із-мислення" "Завдання для виконання"
```

**Модифікатор бюджету** — на обмежених планах знижуйте рівень моделі на один крок:

| План | Фаза планування | Фаза реалізації |
|------|-----------------|-----------------|
| **Max / API (без обмежень)** | Opus 4.7 | Sonnet |
| **Max / API** | Opus 4.6 | Sonnet |
| **Pro / Teams Standard** | Sonnet | Haiku (механічні завдання) |
| **API (обмежений бюджет)** | Sonnet | Haiku |

### Дерево рішень для оптимізації витрат

Високі витрати зазвичай можна виправити. Ця схема допоможе знайти причину та шлях до економії.

```mermaid
flowchart TD
    A([Високі витрати?]) --> B{Контекст<br/>завеликий?}
    B -->|Так| C(Використовуйте /compact<br/>або почніть нову сесію)
    C --> Z([Економія 40-60%<br/>на сесію])

    B -->|Ні| D{Занадто довгі<br/>відповіді?}
    D -->|Так| E(Додайте в CLAUDE.md:<br/>'Будь лаконічним, без пояснень')
    E --> Z2([Економія 20-30%])

    D -->|Ні| F{Постійні<br/>повтори контексту?}
    F -->|Так| G(Перенесіть контекст,<br/>що повторюється, в CLAUDE.md)
    G --> Z3([Економія 15-25%])

    F -->|Ні| H{Хибна модель<br/>для завдання?}
    H -->|Так| I(Haiku для простих завдань)
    I --> Z4([Економія 50-90%<br/>на простих завданнях])

    H -->|Ні| J{Шумний вивід<br/>сервера MCP?}
    J -->|Так| K(Налаштуйте лаконічність MCP<br/>або фільтруйте вивід)
    K --> Z5([Економія 10-20%])

    J -->|Ні| L([Витрати в нормі])

    style A fill:#F5E6D3,color:#333
    style Z fill:#7BC47F,color:#333
    style L fill:#B8B8B8,color:#333

    click A href "../ultimate-guide.uk.md#913-стратегії-оптимізації-витрат" "Високі витрати?"
```

<details>
<summary>ASCII версія</summary>

```
Високі витрати?
├─ Завеликий контекст?   → /compact або нова сесія   (економія 40-60%)
├─ Довгі відповіді?      → CLAUDE.md: будь лаконічним (економія 20-30%)
├─ Повтори контексту?    → Перенесіть у CLAUDE.md    (економія 15-25%)
├─ Хибна модель?         → Haiku для простих завдань  (економія 50-90%)
├─ Шумний MCP?           → Фільтруйте вивід інструм.  (економія 10-20%)
└─ Нічого з цього?       → Витрати в нормі
```

</details>

### Пайплайн стратегій зменшення токенів

Кілька стратегій разом дають кумулятивну економію.

```mermaid
flowchart LR
    BASE([Базовий рівень:<br/>100% токенів]) --> RTK

    subgraph RTK["1: RTK Проксі"]
        R1[Сирий вивід CLI<br/>→ фільтрований вивід]
        R2["git status, cargo test,<br/>pnpm list → стиснуто"]
        R3[Економія 60-90%<br/>на командах CLI]
    end

    RTK --> COMP

    subgraph COMP["2: /compact"]
        C1[Довгий діалог<br/>→ резюме]
        C2[Залишає рішення,<br/>видаляє довгі роздуми]
        C3[Економія 40-60%<br/>на чекпоінті]
    end

    COMP --> CLAUDE_MD

    subgraph CLAUDE_MD["3: CLAUDE.md"]
        CM1[Повторюваний контекст<br/>→ постійні інструкції]
        CM2[Без перепояснень<br/>конвенцій проекту]
        CM3[Економія 15-25%<br/>на сесію]
    end

    CLAUDE_MD --> MODEL

    subgraph MODEL["4: Вибір моделі"]
        MO1[Haiku для простих завдань<br/>замість Sonnet]
        MO2[Та сама якість<br/>за меншу ціну]
        MO3[Економія 50-90%<br/>на простих завд.]
    end

    MODEL --> RESULT([Оптимізовано:<br/>10-20% від бази<br/>для типового використ.])

    style BASE fill:#E85D5D,color:#fff
    style RESULT fill:#7BC47F,color:#333

    click BASE href "../ultimate-guide.uk.md#913-стратегії-оптимізації-витрат" "Базовий рівень"
```
