# Мультиагентні патерни

Патерни для координації декількох екземплярів Claude для паралельної та складної роботи.

---

### Команди агентів — 3 топології оркестрації

Три перевірені топології для координації декількох агентів. Обирайте залежно від незалежності завдань та потреби у спеціалізації.

```mermaid
flowchart TD
    subgraph ORCH["Патерн 1: Оркестратор + Виконавці"]
        OL[Головний оркестратор] --> OW1[Виконавець 1<br/>Frontend]
        OL --> OW2[Виконавець 2<br/>Backend]
        OL --> OW3[Виконавець 3<br/>Тести]
        OW1 & OW2 & OW3 --> OR([Агрегація результатів])
    end

    subgraph PIPE["Патерн 2: Пайплайн (Конвеєр)"]
        PA[Агент A<br/>Вимоги] --> PB[Агент B<br/>Реалізація]
        PB --> PC[Агент C<br/>Рев'ю]
        PC --> PD([Фінальний результат])
    end

    subgraph ROUTE["Патерн 3: Спеціалізований роутер"]
        RR{Агент-роутер<br/>аналізує завдання} --> RC[Код-агент]
        RR --> RT[Тест-агент]
        RR --> RD[Докс-агент]
        RC & RT & RD --> RO([Спеціалізований результат])
    end

    style OL fill:#E87E2F,color:#fff
    style OW1 fill:#6DB3F2,color:#fff
    style OR fill:#7BC47F,color:#333
    style PD fill:#7BC47F,color:#333
    style RR fill:#E87E2F,color:#fff
    style RO fill:#7BC47F,color:#333

    click OL href "../workflows/agent-teams.uk.md" "Головний оркестратор"
```

### Патерн Git Worktree для декількох екземплярів

Git worktrees дозволяють справжню паралельну розробку: кожен екземпляр Claude працює в ізольованій гілці.

```mermaid
flowchart LR
    MB[(Головна гілка<br/>git репозиторій)] --> WA[git worktree add<br/>feature-A]
    MB --> WB[git worktree add<br/>feature-B]
    MB --> WC[git worktree add<br/>bugfix-C]

    WA --> CA[Claude 1<br/>/worktrees/feature-A]
    WB --> CB[Claude 2<br/>/worktrees/feature-B]
    WC --> CC[Claude 3<br/>/worktrees/bugfix-C]

    CA --> CA1([Комміти у feature-A])
    CB --> CB1([Комміти у feature-B])
    CC --> CC1([Комміти у bugfix-C])

    CA1 & CB1 & CC1 --> MERGE([Мердж у main<br/>коли готово])

    style MB fill:#E87E2F,color:#fff
    style CA fill:#6DB3F2,color:#fff
    style MERGE fill:#7BC47F,color:#333

    click MB href "../ultimate-guide.uk.md#912-найкращі-практики-git-та-воркфлоу" "Головна гілка"
```

### Патерн планування з двома екземплярами (Dual-Instance)

Відокремлення планування від виконання запобігає помилкам: Claude-планувальник не має інструментів, тому не може випадково нічого змінити.

```mermaid
sequenceDiagram
    participant U as Користувач
    participant PL as Claude-планувальник<br/>(без інструментів)
    participant EX as Claude-виконавець<br/>(повні інструменти)

    U->>PL: "Сплануй рефакторинг модуля auth"
    Note over PL: Читає докс, аналізує вимоги<br/>Немає ризику виконання — немає інструментів

    PL->>U: Детальний план:<br/>1. Файли для зміни<br/>2. Порядок дій<br/>3. Ризики<br/>4. Стратегія відкату

    U->>U: Ретельний перегляд плану
    Note over U: Чекпоінт людини:<br/>схвалення або правки

    U->>EX: "Виконай цей план: [текст плану]"
    EX->>EX: Реалізація крок за кроком
    EX->>U: Звіти про прогрес + результати

    Note over PL,EX: Ключова ідея: планувальник може бути<br/>ретельнішим без "тривоги" за виконання
```

<details>
<summary>ASCII версія</summary>

```
Користувач → Планувальник (без ікс.): "План X"
               │
      [безпечний аналіз, без ризику]
               │
Планувальник → Користувач: детальний план
               │
Користувач перевіряє + схвалює
               │
Користувач → Виконавець (з інструм.): "Виконай: [план]"
               │
      [реалізація з повним контекстом]
               │
Виконавець → Користувач: результати
```

</details>

### Горизонтальне масштабування (Борис Черний)

Якщо завдання можна паралелізувати, запускайте N екземплярів Claude одночасно.

```mermaid
flowchart LR
    BT([Велике завдання:<br/>Рефактор 50 файлів]) --> DEC{Розподіл на<br/>N підзавдань}

    DEC --> T1["Підзавдання 1<br/>Файли 1-10"]
    DEC --> T2["Підзавдання 2<br/>Файли 11-20"]
    DEC --> TN["Підзавдання N<br/>..."]

    T1 --> CI1[Екземпляр 1]
    T2 --> CI2[Екземпляр 2]
    TN --> CIN[Екземпляр N]

    CI1 & CI2 & CIN --> AGG(Агрегація<br/>результатів)
    AGG --> REV([Рев'ю інтеграції<br/>~в 10 разів швидше])

    style BT fill:#F5E6D3,color:#333
    style DEC fill:#E87E2F,color:#fff
    style CI1 fill:#6DB3F2,color:#fff
    style AGG fill:#B8B8B8,color:#333
    style REV fill:#7BC47F,color:#333

    click BT href "../ultimate-guide.uk.md#917-патерни-масштабування-multi-instance" "Велике завдання"
```

### Матриця прийняття рішень для декількох екземплярів

Не кожне завдання потребує декількох екземплярів. Ця схема допоможе обрати правильний патерн.

```mermaid
flowchart TD
    A([Завдання для виконання]) --> B{Потрібно кілька<br/>екземплярів Claude?}
    B -->|Ні| C([Одна сесія<br/>Стандартне використання])
    B -->|Так| D{Скільки<br/>екземплярів?}
    B -->|"Розподіл плану?"| B2{Треба відокремити<br/>планування?}

    D -->|2-3| E{Потрібна ізоляція<br/>гілок?}
    E -->|Так| F([Git worktrees<br/>Окремі гілки])
    E -->|Ні| G([Кілька терміналів<br/>Один репо])

    D -->|4+| H{Структура завдання?}
    H -->|Незалежні| I([Інструмент Task<br/>Паралельні субагенти])
    H -->|Конвеєр| J([Агентний пайплайн<br/>A → B → C])
    H -->|Різна експертиза| K([Роутер спеціалістів<br/>Маршрутизація за типом])

    B2 --> L([Dual-instance<br/>Планувальник + Виконавець])

    style B fill:#E87E2F,color:#fff
    style C fill:#B8B8B8,color:#333
    style F fill:#7BC47F,color:#333
    style L fill:#6DB3F2,color:#fff

    click B2 href "../workflows/dual-instance-planning.uk.md" "Відокремити планування?"
```
