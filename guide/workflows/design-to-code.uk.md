# Воркфлоу: Від дизайну до коду (Figma MCP)

> **Впевненість**: Рівень 2 — Базується на кейсах Parallel HQ та builder.io. Автоматизація дозволяє скоротити час на верстку компонентів до 75%.

Використання MCP-сервера Figma дозволяє Claude Code зчитувати специфікації безпосередньо з дизайн-макетів, підтримуючи 100% відповідність токенів, відступів та структури компонентів у коді.

---

## 1. Архітектура та ієрархія токенів

Сучасні дизайн-системи використовують трирівневу структуру токенів, яку Claude розуміє при аналізі даних з Figma:

- **Базові (Base)**: Примітивні значення (наприклад, `blue-600: #0066CC`).
- **Композитні (Composite)**: Поєднання примітивів (наприклад, відступи кнопок).
- **Семантичні (Semantic)**: Контекстне значення (наприклад, `primary-action` → `blue-600`).

Claude автоматично мапить змінні з Figma на ваші CSS-змінні або конфігурацію Tailwind.

---

## 2. Основні воркфлоу

### Сценарій А: Фрейм Figma → Продакшн-компонент
Ви надаєте Claude посилання на компонент у Figma. Claude через MCP зчитує стилі, розміри та варіанти, після чого генерує React/TypeScript компонент, що використовує ваші токени та Tailwind класи. Час виконання: 2-3 хвилини.

### Сценарій Б: Аудит розбіжностей (Design Drift)
Claude аналізує вашу папку з компонентами та порівнює її з дизайн-системою у Figma. Він знаходить "магічні числа", жорстко прописані кольори (hex-коди) та пропонує виправити їх на відповідні токени.

### Сценарій В: Візуальне тестування (Figma + Playwright)
За допомогою MCP для Figma та Playwright, Claude може зробити скріншот живого компонента в браузері та порівняти його з еталоном у Figma, вказуючи на візуальні невідповідності (колір, шрифти, інтервали).

---

## 3. Альтернатива: Pencil (IDE-нативне полотно)

[Pencil](https://pencil.dev) (запущено в січні 2026) переносить нескінченне полотно дизайну безпосередньо у ваш IDE (Claude Code / Cursor).
- **Перевага**: Дизайн зберігається у файлах `.pen` (JSON) і версіонується через Git разом із кодом.
- **AI-мультиплеєр**: Кілька агентів можуть працювати над дизайном паралельно.
- **Двостороння робота**: Claude може не тільки читати дизайн, а й вносити зміни в макет.

---

## 4. Типові пастки (Anti-Patterns)

| ❌ Помилка | Чому це погано | ✅ Як правильно |
|-----------|---------------|----------------|
| Ручне переписування | Ризик помилок та розбіжностей | Дозвольте Claude читати Figma через MCP |
| Скріншоти замість специфікацій | Немає даних про токени та логіку | Використовуйте Figma URL для доступу до даних |
| Жорсткі значення в коді | Складність при оновленні бренду | Завжди використовуйте семантичні токени |
