# Ітеративне вдосконалення (Iterative Refinement)

> **Впевненість**: Рівень 2 — Валідований патерн, що спостерігається серед багатьох досвідчених користувачів Claude Code.

Промпт, спостереження, перепромпт до задоволення результатом. Це ядро ефективної розробки за допомогою ШІ.

---

## 1. Основний цикл

1. **Початковий промпт** з чіткою метою.
2. **Claude видає результат**.
3. **Ви оцінюєте** результат за критеріями.
4. **Специфічний фідбек**: "Зміни X, тому що Y".
5. **Повторення** до готовності.

**Ключова думка**: Специфічний фідбек завжди кращий за загальний.

---

## 2. Патерни зворотного зв'язку

| Патерн | Приклад |
|---------|---------|
| **Точна локація** | "Рядок 23: зміни `===` на `==`" |
| **Чітка дія** | "Додай Error Boundary навколо форми" |
| **Обґрунтування** | "Видали `console.log`, бо він розкриває дані користувача" |
| **Пріоритетність** | "Критично: виправ SQL-ін'єкцію. Бажано: додай пагінацію." |

---

## 3. Автономні цикли: Патерн Ральфа Віггама

Ви можете доручити Claude самостійно ітерувати до досягнення критеріїв готовності:

```
Продовжуй покращувати якість коду, поки:
1. Всі тести не пройдуть успішно
2. Зникнуть помилки TypeScript
3. Лінтер показуватиме нуль попереджень

Після кожної ітерації запускай перевірки та виправляй знайдені помилки.
Зупинись, коли всі критерії будуть виконані.
```

---

## 4. Генерація скриптів (Патерн 3-7 ітерацій)

Найбільший ROI (повернення інвестицій) ітеративне вдосконалення дає при створенні скриптів автоматизації:

- **Ітерація 1**: Базова функціональність.
- **Ітерація 2-3**: Обмеження та крайові випадки.
- **Ітерація 4-5**: Зміцнення (обробка помилок, логування).
- **Ітерація 6-7**: Полірування (документація, оптимізація).

---

## 5. Цикл автокорекції при рев'ю (Review Auto-Correction Loop)

Спеціалізований патерн для перевірки PR:
1. **Рев'ю** (виявлення проблем).
2. **Виправлення** (автоматичне застосування правок).
3. **Повторне рев'ю** (перевірка, чи не з'явилися нові баги).
4. **Конвергенція** — зупинка, коли кількість змін стає мінімальною.
