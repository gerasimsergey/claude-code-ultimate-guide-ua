# RPI: Дослідження → План → Реалізація

> **Впевненість**: Рівень 2 — Синтезовано з патернів продуктових команд. Гейтова структура відповідає рекомендаціям Anthropic щодо декомпозиції завдань.

Створюйте складні функції у три фази: спочатку дослідіть технічну можливість (Research), потім складіть план реалізації (Plan), і нарешті — пишіть код (Implement). Кожна фаза завершується створенням документа (артефакту), і перехід до наступної можливий лише після вашого явного схвалення (гейт валідації).

---

## 1. Коли використовувати RPI

- **Технічна можливість невідома**: Ви маєте ідею, але не впевнені, чи спрацює вона технічно.
- **Великий обсяг (Scope)**: Робота займе більше одного дня.
- **Нечіткі вимоги**: Ви знаєте бажаний результат, але не шлях до нього.
- **Високий ризик помилки**: Платежі, безпека, міграції БД, складні інтеграції.

**Пропустіть RPI**, якщо це очевидне виправлення (колір кнопки, опечатка) або дуже дрібна зміна в одному файлі.

---

## 2. Як працюють гейти

1. **Гейт 1 (після Дослідження)**: Ви читаєте `RESEARCH.md` і вирішуєте: **ПРИЙНЯТО (GO)** або **ВІДХИЛЕНО (NO-GO)**. Це запобігає створенню непотрібних або неможливих функцій.
2. **Гейт 2 (після Плану)**: Ви затверджуєте `PLAN.md` ПЕРЕД написанням коду. Правки на цьому етапі безкоштовні.
3. **Тестові гейти (під час Реалізації)**: Кожен крок плану повинен пройти автоматичні тести, перш ніж Claude почне наступний.

---

## 3. Фази воркфлоу

### Фаза 1: Дослідження (RESEARCH.md)
Claude вивчає кодову базу, шукає релевантні бібліотеки та оцінює ризики. Результат — файл з описом того, що вже є, що треба змінити, та точками прийняття архітектурних рішень.

### Фаза 2: План (PLAN.md)
Тільки після отримання "GO", Claude складає детальний план: послідовні кроки, зачеплені файли та конкретні **тестові гейти** для кожного кроку (наприклад, "проходить тест X" або "команда Y видає Z").

### Фаза 3: Реалізація
Claude виконує план крок за кроком. Після кожного кроку запускається тестовий гейт. Якщо він не проходить, Claude зупиняється та повідомляє причину, не намагаючись "імпровізувати" з виправленнями без вашої санкції.

---

## 4. Порівняння воркфлоу

| Воркфлоу | Структура | Найкраще для |
|----------|----------------|------------|
| **RPI** | Research + Plan + Implement | Складні фічі, високий ризик, >1 день роботи |
| **Spec-First** | Spec + Implement | Проекти з чітким дизайном та контрактами API |
| **TDD** | Test-first + Implement | Фокус на якості коду та рефакторингу |
| **Прямий** | Промпт -> Код | Дрібні правки, очевидні завдання (< 2 год) |
