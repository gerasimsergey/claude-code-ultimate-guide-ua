# Рев'ю коду (Code Review)

> **Доступність**: Дослідницьке прев'ю — тільки для планів Teams та Enterprise. Недоступно для Free/Pro акаунтів та організацій із увімкненим Zero Data Retention (ZDR).
> **Запущено**: 9 березня 2026 року

Функція Code Review у Claude Code запускає групу спеціалізованих агентів для аналізу кожного пулл-реквесту в GitHub. Агенти перевіряють зміни (diff) у контексті всієї кодової бази на наявність логічних помилок, вразливостей та регресій.

---

## 1. Як це працює

1. **Тригер**: Відкриття PR, новий пуш або коментар `@claude review`.
2. **Аналіз**: Кілька агентів паралельно аналізують код на інфраструктурі Anthropic.
3. **Верифікація**: Спеціальний крок перевірки відфільтровує помилкові спрацювання (false positives).
4. **Результат**: Коментарі з'являються безпосередньо в PR, ранжовані за рівнем важливості.

### Рівні важливості

| Позначка | Рівень | Значення |
|:-------|:---------|:--------|
| 🔴 | Normal | Баг, який варто виправити перед злиттям |
| 🟡 | Nit | Дрібне зауваження, не блокуюче |
| 🟣 | Pre-existing | Помилка, яка вже була в коді до цього PR |

---

## 2. Конфігурація через REVIEW.md

Додайте файл `REVIEW.md` у корінь репозиторію, щоб встановити власні правила рев'ю:

```markdown
# Правила рев'ю коду

## Завжди перевіряти
- Нові API мають відповідні тести
- Міграції БД є зворотно сумісними
- Помилки не містять конфіденційних внутрішніх даних

## Стиль
- Використовуйте раннє повернення (early returns) замість вкладених умов
- Тільки структуроване логування
```
