# Воркфлоу: Планування з двома інстансами

> **Впевненість**: Рівень 2 — Базується на досвіді практиків (Jon Williams, лютий 2026). Патерн валідований через перехід з Cursor на Claude Code.

Метод полягає у використанні двох інстансів Claude з чітко розділеними ролями: один для планування та рев'ю (**Claude Zero**), інший для впровадження (**Claude One**). Це розділення обов'язків покращує якість архітектури та зменшує кількість помилок при написанні коду.

---

## 1. Коротко про метод

1. **Claude Zero (Планувальник)**: Досліджує код, пише плани, проводить рев'ю. **Ніколи не править код.**
2. **Claude One (Виконавець)**: Зчитує затверджені плани, пише код, робить коміти. **Ніколи не створює плани.**
3. **Людина-контролер**: Затверджує плани перед початком впровадження.
4. **Структура планів**: Папки `Review/` (на перевірці) → `Active/` (в роботі) → `Completed/` (архів).

---

## 2. Коли використовувати цей патерн

### ✅ Використовуйте, якщо:
- **Складні специфікації**: Вимоги потребують уточнення через серію запитань.
- **Критичні функції**: Безпека, платежі, міграції даних.
- **Бюджетні обмеження**: $100-200/міс (це дешевше, ніж багато циклів виправлень в одному вікні).

### ❌ Не використовуйте, якщо:
- **Прості зміни**: Виправлення опечаток, тривіальний рефакторинг.
- **Стислі дедлайни**: Коли швидкість важливіша за якість (і ви готові до циклів виправлень).

---

## 3. Налаштування ролей

### Інстанс 1: Claude Zero (Planner)
Запустіть у першому терміналі та дайте інструкцію:
> "Ти — Claude Zero (Планувальник). Твоя роль: досліджувати код у Plan Mode, писати плани у `.claude/plans/Review/` та перевіряти роботу Claude One. Ти НІКОЛИ не правиш код прямо і НІКОЛИ не робиш коміти."

### Інстанс 2: Claude One (Implementer)
Запустіть у другому терміналі та дайте інструкцію:
> "Ти — Claude One (Виконавець). Твоя роль: зчитувати затверджені плани з `.claude/plans/Active/` та впроваджувати їх точно за специфікацією. Роби коміти після кожного логічного кроку. Ти НІКОЛИ не створюєш плани самостійно."

---

## 4. Аналіз витрат

Хоча ви запускаєте два інстанси, це часто виходить дешевше для складних завдань:
- **Один інстанс**: 1 сесія + 3 цикли виправлень помилок планування = ~$65.
- **Два інстанси**: 1 сесія планування + 1 сесія чистого впровадження = ~$36.
**Економія**: до $30 на одній складній функції.
