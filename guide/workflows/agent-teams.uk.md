# Воркфлоу команд агентів (Agent Teams)

> **Паралельна координація кількох агентів для складних завдань**
> **Статус**: Експериментально (v2.1.32+) | **Модель**: Потрібна Opus 4.6+ | **Прапорець**: `CLAUDE_CODE_EXPERIMENTAL_AGENT_TEAMS=1`

**Що це?** Кілька екземплярів Claude працюють паралельно над спільною кодовою базою, координуючи свої дії автономно. Один сеанс виступає як **Тімлід (Team Lead)**, який розбиває завдання та синтезує результати колег.

**Час читання**: ~30 хв.

---

## 1. Огляд

Команди агентів дозволяють Claude самостійно резервувати завдання, безперервно зливати зміни (continuous merge) та вирішувати конфлікти.

**Ключові характеристики:**
- ✅ **Автономна координація** — Тімлід делегує, колеги спілкуються через поштові скриньки (mailbox).
- ✅ **P2P повідомлення** — Прямий обмін знаннями між агентами.
- ✅ **Git-блокування** — Агенти резервують файли через лок-файли у `.claude/tasks/`.
- ✅ **Ізольований контекст** — Кожен агент має власне вікно в 1 млн токенів.

---

## 2. Архітектура Тімлід — Колеги

```
┌─────────────────────────────────────────────────┐
│         Тімлід (Team Lead - головна сесія)      │
│  - Розбиває завдання на підзавдання             │
│  - Створює сесії колег (Teammates)              │
│  - Синтезує знахідки команди                    │
└─────────────────┬───────────────────────────────┘
                  │
        ┌─────────┴─────────┐
        │                   │
┌───────▼────────┐  ┌───────▼────────┐
│    Колега 1    │◄─┼────────────────►│    Колега 2    │
│ (1M токенів)   │  │   P2P обмін    │ (1M токенів)   │
└────────────────┘  └─────────────────┘────────────────┘
```

**Git-координація:** Агенти працюють у різних гілках/worktree та автоматично пушать/пулять зміни, намагаючись уникнути конфліктів через чітку декомпозицію файлів.

## 3. Налаштування

**Спосіб 1: Змінна оточення**
```bash
export CLAUDE_CODE_EXPERIMENTAL_AGENT_TEAMS=1
claude
```

**Спосіб 2: Файл налаштувань (`~/.claude/settings.json`)**
```json
{
  "env": {
    "CLAUDE_CODE_EXPERIMENTAL_AGENT_TEAMS": "1"
  }
}
```

## 4. Фреймворк прийняття рішень

| Критерій | Команди агентів | Multi-Instance (кілька терміналів) |
|-----------|-------------|----------------|
| **Координація** | Автоматична (Claude) | Ручна (Людина) |
| **Комунікація** | P2P повідомлення | Ручний копіпаст результатів |
| **Когнітивне навантаження** | Низьке (ви — спостерігач) | Високе (ви — оркестратор) |
| **Вартість** | Висока (3x+ токенів) | Середня (2x токенів) |

## 5. Найкращі практики

### Принцип чітких меж (Clear Boundaries)
- **Добре**: Агент 1 — Бекенд API, Агент 2 — Фронтенд компоненти, Агент 3 — Тести.
- **Погано**: Всі три агенти редагують один файл `auth.ts`.

### AGENTS.md для колективного навчання
Створіть файл `AGENTS.md` (аналог `CLAUDE.md`), де ви (як людина) фіксуєте патерни, що спрацювали для агентів, та пастки, яких слід уникати. **Важливо**: ніколи не дозволяйте агентам писати в цей файл самостійно — це призводить до накопичення "сміттєвого" контексту.

### Запобіжники циклів (Loop Guardrails)
Встановіть ліміт ітерацій (наприклад, макс. 8 спроб) для будь-якого підзавдання. Якщо агент застряг — він повинен зупинитися, провести рефлексію ("Що саме пішло не так?") і доповісти тімліду.
