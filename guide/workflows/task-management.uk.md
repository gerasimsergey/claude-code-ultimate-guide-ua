# Воркфлоу: Керування завданнями

**Версія**: Claude Code v2.1.16+
**Час на освоєння**: 15-30 хв

## Огляд

Керування завданнями в Claude Code еволюціонувало від простих списків у пам'яті сесії (`TodoWrite`) до повноцінного **Tasks API**, який зберігає стан на диску (`~/.claude/tasks/`). Це дозволяє продовжувати роботу навіть після закриття термінала або стиснення контексту.

---

## 1. Tasks API vs TodoWrite

| Функція | TodoWrite (Старе) | Tasks API (Нове) |
|---------|-------------------|---------------------|
| **Збереження** | Тільки в пам'яті сесії | На диску (назавжди) |
| **Багатосесійність** | ❌ Втрачається при виході | ✅ Зберігається між сесіями |
| **Залежності** | ❌ Ручне сортування | ✅ Блокування (А блокує Б) |
| **Координація** | Один агент | ✅ Кілька агентів одночасно |

---

## 2. Ключові етапи роботи

### Планування (TaskCreate)
Розбивайте великі фічі на ієрархію завдань. Використовуйте `CLAUDE_CODE_TASK_LIST_ID`, щоб ізолювати список завдань для конкретного проекту:
```bash
export CLAUDE_CODE_TASK_LIST_ID="auth-refactor-2026"
claude
```
Всередині Claude: "Створи ієрархію завдань для рефакторингу авторизації, де тести залежать від впровадження логіки".

### Виконання (TaskList & TaskUpdate)
Слідуйте циклу:
1. `TaskList` — перевірити, що зараз у роботі.
2. `TaskGet` — отримати деталі конкретного завдання.
3. `TaskUpdate` — змінити статус на `in_progress`, а потім на `completed`.

---

## 3. Артефакт безперервності: `progress.md`

Для проектів, що тривають кілька днів, використовуйте файл `progress.md` у корені репозиторію. Це стисле резюме (до 50 рядків) для ШІ, яке він зчитує на початку кожної сесії:
- **active_feature**: ID поточної фічі.
- **last_commit**: хеш останнього коміту.
- **Next action**: чітка наступна дія.

Це дозволяє новому агенту увійти в курс справи за секунди, не перечитуючи всю історію чату.

---

## 4. Поради та обмеження

- **ID завдань**: Завжди використовуйте унікальні ID для кожного репозиторію, щоб уникнути плутанини.
- **Малі завдання**: Не робіть завдання занадто великими (>10 кроків). Краще розбити одну велику таску на кілька дочірніх.
- **Валідація**: Маркуйте завдання як `completed` тільки після успішного проходження тестів.
