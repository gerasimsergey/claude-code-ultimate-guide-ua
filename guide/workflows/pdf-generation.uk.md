# Генерація PDF з Claude Code

> **Впевненість**: Рівень 2 — Базується на перевіреному в продакшені воркфлоу зі стеком Quarto/Typst.

Створюйте професійні PDF-документи (технічну документацію, звіти, whitepapers) за допомогою Claude Code, використовуючи сучасну типографіку та дизайн.

---

## 1. Коротко про стек

Забудьте про складний LaTeX. Ми використовуємо сучасну альтернативу:
- **Quarto**: Головний інструмент для оркестрації та рендерингу.
- **Typst**: Новий двигун типографіки, що миттєво генерує PDF високої якості.
- **Pandoc**: Потужний конвертер Markdown.

---

## 2. Коли використовувати

| Сценарій | Підходить | Альтернатива |
|----------|----------|-------------|
| Технічна документація | ✅ | — |
| Технічні звіти (Whitepapers) | ✅ | — |
| Документація API | ⚠️ | OpenAPI + Redoc |
| Швидкі нотатки | ❌ | Звичайний Markdown |

**Найкраще для**: Об'ємного технічного контенту, який потребує професійної верстки, контролю версій та автоматизації.

---

## 3. Налаштування (Setup)

**macOS**:
```bash
brew install quarto
```

**Перевірка**:
```bash
quarto --version  # Має бути ≥1.4.0
```

---

## 4. Робочий процес

1. **Пишіть контент** у форматі Markdown (`.qmd`).
2. **Додавайте YAML-метадані** (заголовок, автор, дата).
3. **Попередній перегляд**: використовуйте `quarto preview document.qmd` для живого оновлення в браузері.
4. **Рендеринг**: запустіть `quarto render document.qmd` для отримання фінального PDF.

---

## 5. Інтеграція з Claude Code

Ви можете доручити Claude створювати PDF з нуля або конвертувати існуючі README:

**Приклад промпту**:
```
Створи професійну технічну інструкцію для нашого API у форматі Quarto.
Додай зміст, розділи про авторизацію та коди помилок.
Використовуй формат Typst.
```

**Автоматизація через хуки**:
Ви можете налаштувати Claude так, щоб він автоматично перезбирав PDF кожного разу, коли ви редагуєте вихідний `.qmd` файл.
