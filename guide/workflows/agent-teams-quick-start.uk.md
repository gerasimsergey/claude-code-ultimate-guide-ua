# Команди агентів: Швидкий старт

> **Практичне керівництво з використання команд агентів у ваших проектах**
> **Час читання**: 8-10 хв | **Повна документація**: [Команди агентів (Agent Teams)](./agent-teams.md)

Цей посібник допоможе вам швидко налаштувати середовище для роботи з командами агентів та надасть готові патерни для типових інженерних завдань.

---

## 1. Налаштування за 5 хвилин

### Крок 1: Активація функції
Команди агентів наразі є експериментальною функцією. Щоб увімкнути її, встановіть змінну оточення:

```bash
export CLAUDE_CODE_EXPERIMENTAL_AGENT_TEAMS=1
claude
```

### Крок 2: Перевірка
У чаті Claude запитайте: `Are agent teams enabled?`. Ви маєте отримати підтвердження, що функція активна і Claude може створювати групи для паралельної роботи.

### Крок 3: Перший тест
Спробуйте створити просту команду для аналізу вашого README:
```
> Створи команду з 2 агентів для аналізу README:
> - Агент 1: Перевір структуру (розділи, зміст, бейджі)
> - Агент 2: Оціни якість контенту (чіткість, приклади)
```

---

## 2. Патерни для копіювання

### Патерн: Передрелізне рев'ю (3 агенти)
Ідеально підходить для фінальної перевірки проекту перед випуском версії.
- **Аудитор точності**: Перевіряє факти, версії та зовнішні посилання.
- **Контролер консистентності**: Стежить, щоб дані в документації та коді збігалися.
- **Пошук критичних змін**: Аналізує CHANGELOG на наявність breaking changes.

### Патерн: Security PR Review (3 агенти)
Поглиблений аналіз сторонніх пулл-реквестів.
- **Експерт з мови (напр. Rust)**: Перевіряє ідіоматичність коду та обробку помилок.
- **Аудитор безпеки**: Шукає ризики ін'єкцій, витоки токенів та вразливості.
- **Аналітик продуктивності**: Перевіряє алокації та ефективність алгоритмів.

## 3. Матриця прийняття рішень

| Ситуація | Команди агентів? | Чому |
|-----------|---------------|--------|
| **Передрелізне рев'ю** | ✅ ТАК | Потребує паралельних експертних поглядів на різні шари проекту |
| **Виправлення опечатки** | ❌ НІ | Зайві витрати токенів, 1 агент впорається за 10 секунд |
| **Security PR Review** | ✅ ТАК | Високі ставки, потрібно виявити "сліпі плями" в безпеці та Rust |
| **Оновлення документації** | ✅ ТАК | Дозволяє паралельно писати текст, оновлювати індекси та посилання |
| **Лінійне написання коду** | ❌ НІ | Крок Б залежить від А, паралелізація не дасть переваги |

---

## 4. Метрики успіху (ROI)

- **Конвергенція (>50%)**: Якщо два або більше агентів вказують на ту саму проблему, це ознака високої впевненості ШІ в знайденому багу.
- **Унікальні інсайти**: Кожен агент має знайти принаймні одну проблему у своїй вузькій спеціалізації, яку пропустили інші.
- **Економія часу (60-70%)**: Команда агентів має виконувати складний аудит швидше, ніж один агент, який робив би це послідовно.
