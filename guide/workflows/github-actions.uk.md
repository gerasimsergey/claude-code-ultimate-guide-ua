# Воркфлоу GitHub Actions з Claude Code

Автоматизуйте рев'ю коду, сортування ішью та гейти якості, підключивши Claude безпосередньо до вашого GitHub-пайплайну. Використовуйте дві моделі: за згадкою `@claude` (ініційовано людиною) або повністю автономно за подіями Git.

---

## 1. Дві моделі роботи

| Модель | Тригер | Використання |
|-------|---------|----------|
| **Інтерактивна** | Згадка `@claude` у коментарі | Рев'ю за запитом, питання, швидкі виправлення |
| **Автоматизована** | Push, відкриття PR, розклад | Безперервні гейти якості, сортування ішью |

---

## 2. Налаштування (Setup)

Найшвидший спосіб — виконати команду в терміналі Claude Code:
```
/install-github-app
```
Це проведе вас через створення GitHub App, додавання секретів (`ANTHROPIC_API_KEY`) та генерацію базового файлу `claude.yml`.

---

## 3. Патерни автоматизації

### Патерн 1: Рев'ю коду за згадкою @claude
Розробник пише `@claude review this PR` у коментарі до PR, і Claude аналізує зміни та надає конструктивні поради прямо в гілці обговорення.

### Патерн 2: Автоматичне рев'ю при відкритті PR
Кожен новий PR миттєво отримує аудит від Claude. Це дозволяє виявити помилки логіки, проблеми безпеки або пропущене оброблення помилок ще до того, як колега-людина почне перегляд.

### Патерн 3: Сортування ішью (Issue Triage)
Claude читає нові завдання, призначає їм лейбли (bug, enhancement, security), виставляє пріоритет та рекомендує наступні кроки для автора.

### Патерн 4: Перевірка безпеки
Спеціалізований запуск для PR, що зачіпають критичні файли (авторизація, платежі, конфігурації). Claude шукає вразливості, забуті секрети або обходи аутентифікації.

---

## 4. Контроль витрат (Cost Control)

Щоб уникнути непередбачуваних витрат, завжди встановлюйте ліміти:
- Використовуйте параметр `--max-budget-usd 0.50` для кожного запуску.
- Застосовуйте фільтри `paths:`, щоб не запускати Claude для змін у документації.
- Пропускайте чернетки PR (`draft == true`).
