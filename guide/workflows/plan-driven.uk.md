# Розробка на основі плану (Plan-Driven Development)

> **Впевненість**: Рівень 1 — На основі нативної функціональності режиму `/plan` у Claude Code.

Використовуйте режим `/plan` для будь-яких нетривіальних завдань. Claude досліджує кодову базу (в режимі лише для читання), а потім пропонує план реалізації для вашого затвердження.

---

## 1. Воркфлоу /plan

1. **Увійдіть у режим планування**: Натисніть `Shift+Tab` двічі (Звичайний → Авто-прийняття → План) або поставте складне архітектурне питання.
2. **Claude досліджує**: ШІ читає файли, шукає патерни, але НЕ вносить жодних змін.
3. **Claude пише план**: Створюється файл у `.claude/plans/[назва].md`.
4. **Ви перевіряєте**: Читаєте план на предмет повноти, правильності та ризиків.
5. **Затверджуєте та виконуєте**: "План виглядає добре. Виконуй."

---

## 2. Коли використовувати план

| Сценарій | Чому потрібен план |
|----------|-----|
| Зміни у багатьох файлах | Щоб бачити всі зачеплені файли заздалегідь |
| Зміна архітектури | Валідація підходу до написання коду |
| Нові великі функції | Гарантія повної реалізації всіх вимог |
| Незнайома кодова база | Дозвольте Claude спочатку розібратися в структурі |

**Пропускайте план** для виправлення опечаток, поодиноких рядків або додавання коментарів.

---

## 3. Просунутий рівень: Цикл анотування (Патерн Бориса Тане)

Борис Тане (Engineering Lead у Cloudflare) використовує Markdown-плани як **"спільний стан, що змінюється"** між людиною та ШІ.

### Три фази воркфлоу:
1. **ДОСЛІДЖЕННЯ**: Використовуйте "емпатичні" промпти ("Досліди систему авторизації ДУЖЕ глибоко, зверни увагу на всі нюанси..."). Результати записуються в `research.md`.
2. **ПЛАНУВАННЯ (Цикл анотування)**:
   - Claude пише чернетку `plan.md`.
   - Ви відкриваєте файл і додаєте свої коментарі/правки прямо в Markdown (`<!-- Коментар: Використовуй тут інший провайдер -->`).
   - Claude оновлює файл на основі ваших правок.
   - Повторюйте, поки не зникнуть відкриті питання.
3. **РЕАЛІЗАЦІЯ**: Коли план затверджено, він стає "механічним". Claude просто виконує пункти один за одним без прийняття творчих рішень.
