# Майстерність інструментів пошуку: Поєднання rg, grepai, Serena та ast-grep

> **Опануйте мистецтво пошуку коду, поєднуючи правильні інструменти для максимальної ефективності.**

---

## 1. Матриця швидкого вибору

| Мені потрібно... | Інструмент | Приклад команди |
|--------------|---------------|-----------------|
| Знайти точний текст | `rg` (Grep) | `rg "authenticate" --type ts` |
| Знайти за значенням | `grepai` | `grepai search "user login flow"` |
| Знайти визначення функції | `Serena` | `serena find_symbol --name "login"` |
| Знайти структурний патерн | `ast-grep` | `ast-grep "async function $F"` |
| Знайти, хто викликає функцію | `grepai` | `grepai trace callers "login"` |
| Огляд структури файлу | `Serena` | `serena get_symbols_overview` |
| Дослідити проект | `grepai → Serena` | Комбінований патерн |

---

## 2. Порівняння інструментів

| Характеристика | rg (ripgrep) | grepai | Serena | ast-grep |
|---------|--------------|--------|--------|----------|
| **Тип пошуку** | Регулярні вирази | Семантичний | Символьний | Структура AST |
| **Швидкість** | ⚡ ~20ms | 🐢 ~500ms | ⚡ ~100ms | 🕐 ~200ms |
| **Складність** | Низька | Середня | Низька | Висока |
| **Ефективність токенів** | Висока (4x) | Низька | Середня | Середня |

---

## 3. Дерево рішень

### Рівень 1: Що ви знаєте?
- **Знаю ТОЧНИЙ текст?** → Використовуй **rg (ripgrep)**.
- **НЕ знаю точного тексту?** → Переходь на Рівень 2.

### Рівень 2: Яка мета пошуку?
- **Пошук за ЗНАЧЕННЯМ/КОНЦЕПЦІЄЮ** (наприклад, "логіка валідації платежів") → **grepai**.
- **Пошук ВИЗНАЧЕННЯ функції/класу** → **Serena**.
- **Пошук за СТРУКТУРОЮ КОДУ** (наприклад, async без catch) → **ast-grep**.
- **Розуміння ЗАЛЕЖНОСТЕЙ** (хто викликає цю функцію) → **grepai trace**.

---

## 4. Комбінований воркфлоу: Аудит безпеки

**Ціль**: Знайти вразливості в коді.

1. **Семантичний пошук (grepai)**: `grepai search "SQL query construction"` — знаходимо місця роботи з БД.
2. **Структурні патерни (ast-grep)**: `ast-grep 'db.query("${$VAR}")'` — знаходимо потенційні SQL-ін'єкції через шаблонні рядки.
3. **Відстеження залежностей (grepai)**: `grepai trace callers "executeQuery"` — знаходимо всі точки входу, через які дані потрапляють у вразливу функцію.
4. **Точна перевірка (rg)**: `rg "innerHTML" --type ts` — фінальна перевірка на XSS.

---

## 5. Типові пастки

- ❌ **Помилка**: Використання семантичного пошуку для точних збігів (`grepai search "createSession"`). Це повільно і дорого.
- ✅ **Рішення**: Для точних назв завжди використовуйте `rg`.
- ❌ **Помилка**: Рефакторинг без перевірки графа викликів.
- ✅ **Рішення**: Завжди використовуйте `grepai trace callers` перед зміною спільного коду.
