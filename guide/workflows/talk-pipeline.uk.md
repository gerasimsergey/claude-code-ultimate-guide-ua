# Конвеєр підготовки виступів: від ідеї до слайдів з ШІ

> **Впевненість**: Рівень 2 — Валідовано на реальних конференціях (наприклад, DevWithAI Lyon, 2026).

Перетворіть статтю, нотатки або транскрипт у готову доповідь для конференції — включаючи сценарій виступу та промпт для генерації слайдів у Kimi.com. Конвеєр складається з шести етапів із обов'язковим залученням людини для вибору стратегічного напрямку.

---

## 1. Структура конвеєра

1. **Екстракція**: Вилучення головних тем, метрик та структури з ваших матеріалів.
2. **Дослідження (REX)**: (Тільки для виступів на основі досвіду) Аналіз історії Git для підтвердження темпів розробки, кількості комітів та ключових релізів.
3. **Концепції**: Створення каталогу ідей з оцінкою їхньої "цікавості" для аудиторії.
4. **Позиціювання (Чекпоінт)**: Claude пропонує 3-4 варіанти заголовка та кута подачі. **Ви обираєте один**, перш ніж рухатися далі.
5. **Сценарій**: Написання тексту виступу (пітч), розкадровка слайдів та генерація промпту для Kimi.
6. **Ревізія**: Створення "шпаргалок" для Q&A сесії та списку посилань для слухачів.

---

## 2. Передача завдання: Claude → Kimi

На п'ятому етапі Claude генерує файл `{slug}-kimi-prompt.md`. Ви просто копіюєте його вміст і вставляєте у чат на [Kimi.com](https://kimi.com). Kimi автоматично створює презентацію у форматі PowerPoint або PDF, дотримуючись вказаної дизайн-системи (темна тема, акцентні кольори, мінімум тексту).

---

## 3. Чекпоінти: Людина в циклі (Human-in-the-Loop)

Це критично важливий момент. Claude не вибере заголовок за вас. На 4-му етапі система зупиняється і чекає на ваше рішення. Вибір кута подачі (наприклад, "історія успіху" проти "аналіз помилок") визначає тон всього подальшого сценарію.

---

## 4. Чому це працює?

- **Ланцюжки скілів**: Кожен етап зберігає результати у файлах, які зчитує наступний етап. Ви можете зробити перерву в тиждень між плануванням та написанням сценарію.
- **AI-to-AI handoff**: Claude ідеально підходить для структурування наративу, а Kimi — для візуальної генерації. Ми поєднуємо сильні сторони обох систем.
