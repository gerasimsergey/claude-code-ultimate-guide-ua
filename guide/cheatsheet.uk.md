## Основні команди

| Команда | Дія |
|---------|--------|
| `/help` | Контекстна допомога |
| `/powerup` | Інтерактивні анімовані уроки з функцій Claude Code |
| `/clear` | Скинути розмову (очистити історію) |
| `/compact` | Звільнити контекст (стиснути історію) |
| `/status` | Стан сесії + використання контексту |
| `/context` | Детальна розбивка токенів |
| `/plan` | Увійти в режим планування (Plan Mode) — без внесення змін |
| `/ultraplan` | Хмарний режим планування — чернетка в хмарі, перегляд у браузері (v2.1.91+) |
| `/execute` | Вийти з режиму планування (застосувати зміни) |
| `/model` | Змінити модель (sonnet/opus/opusplan) |
| `/insights` | Аналітика використання + звіт про оптимізацію |
| `/simplify` | Виявити надмірну складність (over-engineering) у зміненому коді + автовиправлення |
| `/batch` | Масштабний рефакторинг через 5–30 паралельних агентів у worktree |
| `/teleport` | Телепортувати сесію з вебу |
| `/tasks` | Моніторинг фонових завдань |
| `/remote-env` | Налаштувати хмарне середовище |
| `/remote-control` | Запустити сесію дистанційного керування (Research Preview, Pro/Max) |
| `/rc` | Аліас для /remote-control |
| `/mobile` | Отримати посилання на завантаження мобільного додатка Claude |
| `/fast` | Перемкнути режим fast (швидкість 2.5x, вартість 6x) |
| `/voice` | Перемкнути голосове введення (утримуйте Space, щоб говорити, відпустіть, щоб надіслати) |
| `/recap` | Резюме контексту сесії при поверненні після перерви (v2.1.108) |
| `/effort [level]` | Глибина мислення: low/medium/high/xhigh/max; без аргументів = інтерактивний повзунок (v2.1.111) |
| `/tui [fullscreen]` | Повноекранний TUI рендеринг без мерехтіння (v2.1.110) |
| `/focus` | Перемкнути мінімалістичний режим фокусу, окремо від Ctrl+O (v2.1.110) |
| `/less-permission-prompts` | Сканувати транскрипти та запропонувати білий список інструментів тільки для читання (v2.1.111) |
| `/btw [question]` | Спливаюче вікно для побічного запитання — ефемерний агент тільки для читання, без впливу на історію та інструменти |
| `/loop [interval] [prompt]` | Повторюваний запуск промпту (напр: `/loop 5m check the deploy`, за замовчуванням 10m) |
| `/stats` | Графік використання, улюблена модель, стрік *(аліас для `/usage` з v2.1.118)* |
| `/usage` | Використання токенів + вартість за кожною моделлю (v2.1.118) |
| `/ultrareview` | Багатоагентне хмарне рев'ю коду (v2.1.114) |
| `/rename [name]` | Назвати або перейменувати поточну сесію |
| `/copy` | Інтерактивний вибір для копіювання блоку коду або всієї відповіді |
| `/debug` | Систематичне усунення несправностей |
| `/exit` | Вийти (або Ctrl+D) |

```

4. Опис завдання       → Чіткий, конкретний промпт

9. /compact            → Коли контекст >70%
```

## Управління контекстом (КРИТИЧНО)

### Статус-рядок (Statusline)

```
Model: Sonnet | Ctx: 89.5k | Cost: $2.11 | Ctx(u): 56.0%
```
**Стежте за `Ctx(u):`** → >70% = `/compact`, >85% = `/clear`

**Покращений статус-рядок ([ccstatusline](https://github.com/sirmalloc/ccstatusline)):** Додайте до `~/.claude/settings.json`:
```json
{ "statusLine": { "type": "command", "command": "npx -y ccstatusline@latest", "padding": 0 } }
```

### Пороги контексту

| % Контексту | Статус | Дія |
|-----------|--------|--------|
| 0-50% | Зелений | Працюйте вільно |
| 50-70% | Жовтий | Будьте вибірковими |
| 70-90% | Помаранчевий | `/compact` зараз |
| 90%+ | Червоний | Необхідно `/clear` |

### Дії за симптомами

| Ознака | Дія |
|------|--------|
| Короткі відповіді | `/compact` |
| Часта забудькуватість | `/clear` |
| >70% контексту | `/compact` |
| Завдання виконано | `/clear` |

### Команди відновлення контексту

| Команда | Використання |
|---------|-------|
| `/compact` | Підсумувати та звільнити контекст |
| `/clear` | Почати з чистого аркуша |
| `/rewind` | Скасувати останні зміни |
| `claude -c` | Продовжити останню сесію (CLI прапорець) |
| `claude -r <id>` | Продовжити конкретну сесію (CLI прапорець) |

## Під капотом (Швидкі факти)

| Концепція | Ключовий момент |
|---------|-----------|
| **Master Loop** | Простий `while(tool_call)` — без DAG або класифікаторів |
| **Інструменти** | 8 основних: Bash, Read, Edit, Write, Grep, Glob, Task, TodoWrite |
| **Контекст** | ~200K токенів, авто-стиснення при 75-92% |
| **Субагенти** | Ізольований контекст, макс. глибина=1 |
| **Філософія** | "Менше каркасів, більше моделі" — довіра до міркувань Claude |

**Глибоке занурення**: [Архітектура та нутрощі](./core/architecture.md)

## Режим планування та мислення

| Функція | Активація | Використання |
|---------|------------|-------|
| **Plan Mode** | `Shift+Tab × 2` або `/plan` | Дослідження без модифікацій |
| **OpusPlan** | `/model opusplan` | Opus для планування, Sonnet для виконання |
| **Ultraplan** | `/ultraplan <prompt>` | Хмарне планування, перегляд у браузері, вільний термінал |

> **Opus 4.7** (v2.1.114+): Рівень зусиль (effort) за замовчуванням = **xhigh**. Новий рівень `xhigh` знаходиться між `high` та `max` — точніший контроль міркувань/затримки. Використовуйте `ultrathink` для форсування максимальних зусиль.

| Контроль | Дія | Персистентність |
|---------|--------|-------------|
| **Alt+T** | Перемкнути мислення on/off | Сесія |
| **/config** | Увімкнути/вимкнути глобально | Постійно |
| **Слайдер `/model`** | Стрілки вліво/вправо: `low\|medium\|high\|xhigh` | Сесія |
| **`CLAUDE_CODE_EFFORT_LEVEL`** | Змінна оточення | Сесія оболонки |
| **Налаштування `effortLevel`** | В settings.json | Постійно |
| **`effort` у frontmatter скіла** | Перевизначення для скіла | На кожен виклик |

**Порада щодо вартості**: Для простих завдань вимикайте мислення (Alt+T) → швидше та дешевше.

**Воркфлоу OpusPlan**: `/model opusplan` → `Shift+Tab × 2` (план з Opus) → `Shift+Tab` (виконання з Sonnet)

## Золоті правила

1. **Завжди переглядайте diff** перед прийняттям.
2. **Використовуйте `/compact`**, перш ніж контекст стане критичним (>70%).
3. **Будьте конкретними** у запитах (ЩО, ДЕ, ЯК, ПЕРЕВІРКА).
4. **Спочатку режим планування** для складних або ризикованих завдань.
5. **Створюйте CLAUDE.md** для кожного проекту.
6. **Часто робіть коміти** після кожного виконаного завдання.
7. **Знайте, що надсилається** — промпти, файли, результати MCP → Anthropic.

## Дерево швидких рішень

```

```
