## Основні команди

| Команда | Дія |
|---------|--------|
| `/help` | Контекстна допомога |
| `/powerup` | Інтерактивні анімовані уроки функцій Claude Code |
| `/clear` | Скинути розмову |
| `/compact` | Звільнити контекст (стиснути) |
| `/status` | Стан сесії + використання контексту |
| `/context` | Детальний розбір токенів |
| `/plan` | Увійти в режим планування (без змін) |
| `/ultraplan` | Хмарний режим планування — чернетка в хмарі, перегляд у браузері |
| `/execute` | Вийти з режиму планування (застосувати зміни) |
| `/model` | Змінити модель (sonnet/opus/opusplan) |
| `/insights` | Аналітика використання + звіт з оптимізації |
| `/simplify` | Виявлення "овер-інжинірингу" в коді + авто-виправлення |
| `/batch` | Масштабні рефакторинги через 5–30 паралельних агентів |
| `/teleport` | Телепортувати сесію з вебу |
| `/tasks` | Моніторинг фонових завдань |
| `/remote-control` | Почати сесію дистанційного керування (Pro/Max) |
| `/rc` | Аліас для /remote-control |
| `/mobile` | Отримати посилання на мобільний додаток Claude |
| `/fast` | Увімкнути швидкий режим (2.5x швидкість, 6x вартість) |
| `/voice` | Увімкнути голосове введення (Space — говорити) |
| `/effort [level]` | Глибина міркувань (Thinking): low/medium/high/xhigh/max |
| `/tui` | Повноекранний TUI режим (без мерехтіння) |
| `/usage` | Використання токенів та вартість за моделями |
| `/copy` | Інтерактивний вибір блоку коду для копіювання |
| `/debug` | Систематичне усунення несправностей |
| `/exit` | Вийти (або Ctrl+D) |

## Маловідомі функції (Але офіційні!)

| Функція | Опис |
|---------|--------------|
| **Tasks API** | Списки завдань із залежностями, що зберігаються між сесіями |
| **Background Agents** | Під-агенти працюють у фоні, поки ви пишете код |
| **Agent Teams** | Координація декількох агентів (TeamCreate/SendMessage) |
| **Auto-Memories** | Автоматичне захоплення контексту між сесіями (MEMORY.md) |
| **LSP Tool** | Навігація як в IDE: символи, типи, посилання. 11 мов. Дуже швидко |
| **Remote Control** | Керування локальною сесією з телефону або браузера |
| **`/loop`** | Планувальник завдань у межах сесії: `/loop 5m check the deploy` |
| **Output Styles** | Стилі виводу: **Default** (стисло), **Explanatory** (з обґрунтуванням), **Learning** (навчальний режим) |

## Типовий робочий процес

1. **Початок** → `claude`
2. **Контекст** → `/status`
3. **План** → `Shift+Tab × 2` (для складних завдань)
4. **Завдання** → Чіткий та конкретний промпт
5. **Рев'ю** → **Завжди читайте diff!**
6. **Вердикт** → Прийняти (y) або Відхилити (n)
7. **Перевірка** → Запуск тестів
8. **Коміт** → Після завершення логічного етапу
9. **Стиснення** → `/compact`, якщо контекст >70%

## Керування контекстом (КРИТИЧНО)

**Статус-рядок**: `Ctx(u): 56.0%`
- **>70%** = `/compact`
- **>85%** = `/clear`

| Контекст % | Статус | Дія |
|-----------|--------|--------|
| 0-50% | Зелений | Працюйте вільно |
| 50-70% | Жовтий | Будьте вибірковими |
| 70-90% | Помаранчевий | `/compact` негайно |
| 90%+ | Червоний | Обов'язково `/clear` |

## Вибір моделі та Міркування

| Завдання | Модель | Глибина міркувань |
|------|-------|--------|
| Тести, бойлерплейт, назви | Haiku | low |
| Розробка фіч, дебаг | Sonnet | medium–high |
| Архітектура, аудит безпеки | Opus | high–max |

**Порада**: Для простих завдань вимикайте Thinking (`Alt+T`) — це швидше і дешевше.

## Формула швидкого промпту

- **ЩО**: Конкретний результат
- **ДЕ**: Шляхи до файлів/модулів
- **ЯК**: Обмеження, підхід, бібліотеки
- **ПЕРЕВІРКА**: Критерії успіху

**Приклад**: "Додай валідацію в src/login.ts. ЯК: Zod. ПЕРЕВІРКА: помилка при невірному email".

---

## Золоті правила

1. **Завжди перевіряйте diff** перед натисканням `y`.
2. **Стискайте контекст** (`/compact`) до критичного рівня.
3. **Будьте специфічними**: Файли, рядки, типи.
4. **Plan Mode спочатку** для ризикованих змін.
5. **Створюйте CLAUDE.md** для кожного проекту.
6. **Комітьте часто** — після кожного виконаного кроку.
