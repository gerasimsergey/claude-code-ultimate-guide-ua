## Серія 2.1.x (Січень-Травень 2026)

### v2.1.131 (2026-05-06)

> Виправлення активації розширення VS Code на Windows, виправлення авторизації Mantle.

- **Виправлено**: Помилка активації розширення VS Code на Windows через жорстко прописаний шлях збірки у вбудованому SDK (баг поліфілу `createRequire`).
- **Виправлено**: Помилка авторизації кінцевої точки Mantle через відсутність заголовка `x-api-key`.

---

### v2.1.129 (2026-05-06)

> Прапорець --plugin-url, CLAUDE_CODE_PACKAGE_MANAGER_AUTO_UPDATE, відновлення історії Ctrl+R, виправлення skillOverrides, понад 20 виправлень помилок.

- **Нове**: Прапорець `--plugin-url <url>` для завантаження `.zip` архіву плагіна за URL для поточної сесії.
- **Нове**: Змінна оточення `CLAUDE_CODE_FORCE_SYNC_OUTPUT=1` для примусового ввімкнення синхронізованого виводу в терміналах, де автовизначення не працює (наприклад, Emacs `eat`).
- **Нове**: `CLAUDE_CODE_PACKAGE_MANAGER_AUTO_UPDATE`: якщо встановлено, інсталяції Homebrew або WinGet запускають команду оновлення у фоновому режимі та пропонують перезапуск.
- **Змінено**: Оголошення `themes` та `monitors` у маніфесті плагіна тепер мають бути у `"experimental": { ... }` — верхній рівень все ще працює, але `claude plugin validate` видаватиме попередження.
- **Покращено**: Вибір історії Ctrl+R знову за замовчуванням показує всі проекти (поведінка до v2.1.124) — натисніть Ctrl+S, щоб звузити до поточного проекту або сесії.
- **Покращено**: Сторонні хмарні провайдери (Bedrock, Vertex, Foundry, шлюз `ANTHROPIC_BASE_URL`) більше не бачать підказок спіннера, що вказують на власні ресурси Anthropic.
- **Виправлено**: Налаштування `skillOverrides` тепер працює: `off` приховує від моделі та `/`, `user-invocable-only` приховує лише від моделі, `name-only` згортає опис.

---

### v2.1.126 (2026-05-01)

> Список моделей /v1/models у виборі /model, очищення проекту claude, вставка коду OAuth для WSL2/SSH, PowerShell 7 на Windows, безпека, понад 40 виправлень помилок.

- **Нове**: Вибір `/model` тепер показує моделі з вашого шлюзу через кінцеву точку `/v1/models`, якщо `ANTHROPIC_BASE_URL` вказує на сумісний шлюз.
- **Нове**: Команда `claude project purge [шлях]` видаляє весь стан Claude Code для проекту (транскрипти, завдання, історію файлів, записи конфігурації).
- **Покращено**: На Windows тепер виявляється PowerShell 7 (з Microsoft Store, MSI або як .NET інструмент); якщо інструмент PowerShell увімкнено, Claude використовує його як основну оболонку.
- **Безпека**: Виправлено ігнорування `allowManagedDomainsOnly` / `allowManagedReadPathsOnly`, коли джерело керованих налаштувань з вищим пріоритетом не мало блоку `sandbox`.
