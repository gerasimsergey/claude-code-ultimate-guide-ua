# Відомі проблеми та критичні помилки

Цей документ відстежує підтверджені критичні проблеми, що впливають на користувачів Claude Code, на основі звітів спільноти та офіційних повідомлень.

> **Останнє оновлення**: 23 квітня 2026
> **Джерело**: [GitHub Issues](https://github.com/anthropics/claude-code/issues) + Офіційні повідомлення Anthropic

---

## 🚨 Активні критичні проблеми

### 0. Баги кешу промптів — Тихе роздуття витрат (Березень 2026 - дотепер)

**Критичність**: 🔴 **ВИСОКА - ВПЛИВ НА ВИТРАТИ**
**Статус**: ⚠️ ЧАСТКОВО ВИПРАВЛЕНО (Баги 2 та 3 все ще активні у v2.1.88)
**Проблема**: [#40524](https://github.com/anthropics/claude-code/issues/40524)

#### Суть проблеми
Три незалежні баги порушують роботу префіксного кешування Anthropic, змушуючи систему платити за створення кешу (`cache_creation`, повна вартість) замість читання (`cache_read`, знижена вартість).

-   **Баг 3 (заголовок атрибуції)**: 2-5-кратне роздуття вартості системного промпту (~12K токенів) при кожному старті сесії та виклику субагента.
-   **Баг 2 (resume + 10+ скілів)**: повна перебудова кешу на 87-118K токенів при кожному відновленні сесії. У найгірших випадках це призводить до **10-20-кратної вартості кожного ходу**.

**Тимчасове рішення для Багу 3** (рекомендується застосувати негайно):
Додайте в `~/.claude/settings.json`:
```json
{
  "env": {
    "CLAUDE_CODE_ATTRIBUTION_HEADER": "false"
  }
}
```

**Тимчасове рішення для Багу 2**: Уникайте команд `--resume` та `--continue`. Починайте нові сесії, поки не вийде виправлення.

### 1. Автоматичне створення GitHub-іш'ю не в тому репозиторії (Грудень 2025 - дотепер)

**Критичність**: 🔴 **КРИТИЧНА - РИЗИК ПРИВАТНОСТІ ТА БЕЗПЕКИ**
**Статус**: ⚠️ АКТИВНО (станом на 28 січня 2026)
**Проблема**: [#13797](https://github.com/anthropics/claude-code/issues/13797)

#### Суть проблеми
Claude Code **систематично створює GitHub-іш'ю у публічному репозиторії `anthropics/claude-code`** замість приватного репозиторію користувача, навіть якщо ви працюєте в локальній директорії свого проекту.

#### Наслідки
**ВИСОКІ - ПРИВАТНІСТЬ/БЕЗПЕКА**: Підтверджено понад 17 випадків, коли користувачі випадково розкривали конфіденційну інформацію в публічному доступі (схеми БД, облікові дані API, архітектуру інфраструктури).

**Заходи безпеки**:
1.  **Завжди перевіряйте цільовий репозиторій** перед підтвердженням команди `gh`.
2.  Явно вказуйте репозиторій: `gh issue create --repo USER/REPO`.
3.  Використовуйте режим ручного підтвердження для всіх команд `gh`.

### 2. Надмірне споживання токенів (Січень 2026 - дотепер)

**Критичність**: 🟠 **ВИСОКА - ВПЛИВ НА ВИТРАТИ**
**Статус**: ⚠️ ПОВІДОМЛЕНО (Anthropic проводить розслідування)
**Проблема**: [#16856](https://github.com/anthropics/claude-code/issues/16856)

Користувачі повідомляють про **в 4 рази швидше споживання токенів** порівняно з попередніми версіями. Ліміти, яких раніше вистачало на тиждень, вичерпуються за 1-2 дні.

**Тимчасові поради**:
-   Частіше запускайте `/clear` (після досягнення 50-60% контексту).
-   Розбивайте складні завдання на кілька коротких сесій.
-   Використовуйте субагентів для ізольованих завдань.

## ✅ Вирішені історичні проблеми

### Інцидент "Потрійної упряжі" (Березень-Квітень 2026)
**Статус**: ✅ **ВИРІШЕНО** (20 квітня 2026)
Три незалежні зміни в системних промптах погіршили якість коду на 6 тижнів. Anthropic відкотив зміни та reset-нув ліміти використання для всіх підписників.

### Деградація якості моделі (Серпень-Вересень 2025)
**Статус**: ✅ **ВИРІШЕНО**
Проблема була пов'язана не з моделлю, а з інфраструктурними багами (неправильна маршрутизація трафіку та помилки компіляції XLA:TPU).
