# Навчання програмуванню з ШІ: Посібник свідомого розробника

> **Впевненість**: Рівень 2 — На основі академічних досліджень (2023-2025) та відгуків освітян.
> **Аудиторія**: Джуніор-розробники, студенти CS, випускники буткемпів.
> **Час читання**: ~15 хвилин.

---

## 1. Проблема за 60 секунд

ШІ може зробити вас у 3 рази продуктивнішим АБО непридатним для працевлаштування через 3 роки. Різниця лише в тому, як ви його використовуєте.

**ШІ — це ваш GPS:**
- Чудово підходить для того, щоб швидко кудись дістатися.
- Небезпечно, якщо ви втрачаєте здатність орієнтуватися без нього.
- Дійсно корисно, коли ви розумієте карту ТА використовуєте GPS.

## 2. Пастка "Вайб-кодингу" (Vibe Coding)

Термін, введений Андрієм Карпати (Feb 2025): написання коду "повністю віддаючись вайбам", без розуміння того, що саме генерує модель.

**Симптоми:**
- "Прийняти все" (Accept All) без читання диффів.
- Копіпаст помилок без розуміння першопричини.
- Налагодження через прохання ШІ робити випадкові зміни, поки "не запрацює".

Це призводить до **Боргу розуміння (Comprehension debt)**: прірви між обсягом коду в системі та рівнем його розуміння людиною.

---

## 3. Протокол UVAL

Системний підхід до використання ШІ без втрати кваліфікації.

| Крок | Дія | Навіщо це потрібно |
|------|--------|----------------|
| **U** (Understand) | Зрозумій спочатку | Щоб ставити кращі питання та ловити помилки |
| **V** (Verify) | Перевір | Щоб переконатися, що ви навчилися, а не просто скопіювали |
| **A** (Apply) | Застосуй | Перетворення знань на навичку через модифікацію |
| **L** (Learn) | Вивчи | Закріплення інсайтів для довгострокового запам'ятовування |

## 4. Шкала зрілості впровадження агентів (0-6)

Ніколя Мартіньоль (Principal Engineer, Back Market) запропонував таку шкалу в березні 2026:

| Рівень | Профіль | Сигнал |
|-------|---------|--------|
| **0** | Без ШІ | Використання лише чат-ботів у браузері |
| **1** | Автодоповнення | Copilot, Cursor — без рівня агентів |
| **2** | Копіпаст | ChatGPT/Claude у браузері -> ручний копіпаст |
| **3** | Користувач Claude Code | Режим Plan, прості промпти, ручне рев'ю |
| **4** | Делегатор етапів | Передача цілих етапів (дослідження, архітектура, тести) |
| **5** | Контекст-інженер | Проектування `CLAUDE.md`, кастомні скіли та MCP-сервери |
| **6** | Оркестратор | Координація графів агентів та циклів навчання |

## 5. Поради для техлідів та менеджерів

Швидкість (Velocity) — це оманливий показник. Вона не показує прогалини в знаннях, що утворюються під капотом.

**Метрики, що показують реальний ріст:**
- **Здатність пояснити код на рев'ю**: Запитайте "проведи мене через свій підхід". Відповідь "так запропонував ШІ" — це червоний прапор.
- **Самостійне налагодження**: Час, за який джун вирішує блокери без ШІ.
- **Предикція результатів**: Запитайте "що зробить цей код?" до його запуску.

---

## 6. Регуляторні ризики (Закон ЄС про ШІ)

Для команд, що працюють у медицині, фінансах або держсекторі, "борг розуміння" — це ризик комплаєнсу.
**Закон ЄС про ШІ (EU AI Act)** вимагає "значущого людського нагляду". Якщо розробник не може пояснити логіку ШІ-генерованого коду, система може бути визнана такою, що не відповідає стандартам безпеки.
