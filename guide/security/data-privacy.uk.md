# Посібник із конфіденційності та зберігання даних

> **Важливо**: Усе, чим ви ділитеся з Claude Code, надсилається на сервери Anthropic. Цей посібник пояснює, які саме дані залишають вашу машину та як захистити конфіденційну інформацію.

---

## TL;DR - Підсумок зберігання даних

| Конфігурація | Термін зберігання | Навчання ШІ | Як увімкнути |
|---------------|------------------|----------|---------------|
| **Споживчий план (за замовчуванням)** | 5 років | Так | (стан за замовчуванням) |
| **Споживчий план (з відмовою)** | 30 днів | Ні | [claude.ai/settings](https://claude.ai/settings/data-privacy-controls) |
| **Team / Enterprise / API** | 30 днів | Ні (за замовчуванням) | Використовуйте корпоративні плани або API-ключі |
| **ZDR (Нульове зберігання)** | 0 днів на сервері | Ні | Належним чином налаштовані API-ключі |

**Негайна дія**: [Вимкніть використання даних для навчання](https://claude.ai/settings/data-privacy-controls), щоб скоротити термін зберігання з 5 років до 30 днів.

---

## 1. Розуміння потоку даних

### Що залишає вашу машину

Коли ви використовуєте Claude Code, наступні дані надсилаються до Anthropic:
- Промпти, які ви вводите.
- Вміст файлів, які читає Claude (включаючи `.env`, якщо вони не виключені!).
- Результати роботи MCP-серверів (SQL-запити, відповіді API).
- Вивід команд Bash.
- Повідомлення про помилки та стеки викликів (stack traces).

---

## 2. Політики зберігання Anthropic

### Рівень 1: Споживчий за замовчуванням (Навчання увімкнено)
- **Зберігання**: 5 років.
- **Використання**: Покращення моделей, дані для навчання.
- **Для кого**: Безкоштовні, Pro, Max плани з увімкненим налаштуванням навчання.

### Рівень 2: Відмова від навчання (Opt-Out)
- **Зберігання**: 30 днів.
- **Використання**: Тільки моніторинг безпеки та запобігання зловживанням.
- **Як увімкнути**: Вимкніть "Allow model training on your conversations" у налаштуваннях конфіденційності на claude.ai.

### Рівень 4: Нульове зберігання даних (Zero Data Retention — ZDR)
- **Зберігання**: 0 днів на сервері.
- **Вимоги**: Спеціально налаштовані API-ключі.
- **Кейси**: HIPAA, GDPR, PCI-DSS, державні контракти.

---

## 3. Відомі ризики

### Ризик 1: Автоматичне читання файлів
За замовчуванням Claude може прочитати `.env`, `.env.local`, `credentials.json` або ключі SSH, якщо вони знаходяться в робочій директорії.
**Захист**: Використовуйте `permissions.deny` у `settings.json`.

### Ризик 2: Команда /bug надсилає ВСЕ (зберігання 5 років)
Коли ви запускаєте `/bug`, ваша **повна історія розмов** (включаючи код і секрети) надсилається для аналізу. Ці дані зберігаються **5 років** незалежно від ваших налаштувань навчання чи плану.
**Захист**: Вимкніть команду за допомогою `export DISABLE_BUG_COMMAND=1`.

### Ризик 3: Тихе встановлення Native Messaging Host (Квітень 2026)
Claude Desktop встановлює маніфести нативних повідомлень у всі Chromium-браузери (Chrome, Brave, Arc тощо) без явного запиту користувача для роботи функції "Claude in Chrome".
**Захист**: Перевірте директорію `NativeMessagingHosts` та видаліть файл `com.anthropic.claude_browser_extension.json`, якщо ви не використовуєте цю функцію.

## 4. Захисні заходи

### 4.1 Конфігурація виключень
Використовуйте `permissions.deny` у `.claude/settings.json`:

```json
{
  "permissions": {
    "deny": [
      "Read(./.env*)",
      "Edit(./.env*)",
      "Write(./.env*)",
      "Bash(cat .env*)",
      "Read(./secrets/**)",
      "Read(./**/*.pem)",
      "Read(./**/*.key)"
    ]
  }
}
```

### 4.2 Вимкнення телеметрії та звітів про помилки
Додайте ці змінні у ваш профіль оболонки (`~/.zshrc` або `~/.bashrc`):
```bash
export DISABLE_TELEMETRY=1          # Вимикає метрики Statsig
export DISABLE_ERROR_REPORTING=1    # Вимикає звіти Sentry
export DISABLE_BUG_COMMAND=1        # Вимикає команду /bug
```

## 5. Конституційний ШІ (Constitutional AI)

Anthropic використовує "Конституцію" для керування поведінкою Claude. Ієрархія цінностей при конфліктах:
1. **Безпека** — ніколи не компрометувати людський контроль.
2. **Етика** — чесність та уникнення шкоди.
3. **Комплаєнс** — внутрішні вказівки та політики.
4. **Корисність** — реальна допомога користувачу.

Це означає, що у критичних питаннях безпеки Claude обере відмову або обережність, навіть якщо це зробить його "менш корисним" у цей момент.
