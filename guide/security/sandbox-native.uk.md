# Нативна пісочниця в Claude Code

> **Впевненість**: Рівень 1 — Офіційна документація Anthropic.
> **Час читання**: ~15 хвилин.

---

## Коротко (TL;DR)

Claude Code включає вбудовану **нативну пісочницю** (v2.1.0+), що використовує примітиви ОС для ізоляції команд bash:

| Аспект | Деталі |
|--------|---------|
| **macOS** | Seatbelt (вбудовано, працює одразу) |
| **Linux/WSL2** | bubblewrap + socat (потрібно встановити) |
| **Файлова система** | Читання всього (налаштовується), запис лише в проект |
| **Мережа** | SOCKS5 проксі, білі/чорні списки доменів |
| **Режими** | Авто-дозвіл (bash без запиту) vs Звичайні дозволи |
| **Обхід (Escape)** | `dangerouslyDisableSandbox` для несумісних утиліт |

---

## 1. Чому нативна пісочниця?

Система дозволів Claude Code створює дилему:
- **DSP (пропуск дозволів)** — швидко, автономно, але небезпечно.
- **Інтерактивні дозволи** — безпечно, але повільно для великих рефакторингів.

**Нативна пісочниця вирішує це**: дозволяє Claude вільно працювати всередині встановлених ОС кордонів. Пісочниця стає периметром безпеки, а не система дозволів.

---

## 2. Примітиви ОС

### macOS: Seatbelt
Працює "з коробки". Використовує фреймворк **macOS Sandbox** (TrustedBSD Mandatory Access Control) для фільтрації системних викликів на рівні ядра.

### Linux/WSL2: bubblewrap
Потребує встановлення `bubblewrap` та `socat`. Використовує простори імен (namespaces) ядра Linux та seccomp-bpf для створення ізольованого середовища для кожної команди.

```bash
# Встановлення для Ubuntu/Debian
sudo apt-get install bubblewrap socat
```

---

## 3. Ізоляція файлової системи

### Політика за замовчуванням
- **Доступ на читання**: Весь комп'ютер (крім явно заборонених папок).
- **Доступ на запис**: Лише поточна робоча директорія (CWD) та її підпапки.

Це дозволяє Claude аналізувати проект та залежності, але запобігає випадковій модифікації системних файлів.

---

## 4. Мережева ізоляція

Усі мережеві з'єднання з пісочниці проходять через **SOCKS5 проксі**, який перевіряє домени за списками дозволених.

**Приклад конфігурації**:
```json
{
  "sandbox": {
    "network": {
      "policy": "deny",
      "allowedDomains": [
        "api.anthropic.com",
        "*.npmjs.org",
        "github.com"
      ]
    }
  }
}
```

---

## 5. Механізм обходу (Escape Hatch)

Деякі інструменти (наприклад, `docker`, `kubectl`) несумісні з пісочницею, бо потребують доступу до Unix-сокетів або специфічних API ядра.
Для них Claude може запросити виконання **поза пісочницею** через параметр `dangerouslyDisableSandbox` (з відповідним запитом до користувача).

---

## 6. Обмеження та ризики безпеки

1. **Доменний фронтинг (Domain Fronting)**: Використання CDN (наприклад, Cloudflare) для завантаження шкідливого коду через "довірені" домени.
2. **Unix-сокети**: Надання доступу до `/var/run/docker.sock` дозволяє агенту повністю захопити хост-машину.
3. **Ескалація через права запису**: Ніколи не дозволяйте запис у папки з `$PATH` (наприклад, `/usr/local/bin`).

---

## 7. Приклади конфігурацій

### Суворий режим (CI/CD, невідомий код)
```json
{
  "sandbox": {
    "autoAllowMode": true,
    "allowUnsandboxedCommands": false,
    "network": { "policy": "deny", "allowedDomains": ["api.anthropic.com", "github.com"] }
  }
}
```
