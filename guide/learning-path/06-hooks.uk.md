# Модуль 06: Хуки та події

**Час**: 1 година | **Складність**: ⭐⭐ Середній

## Мета

Автоматизувати відповіді на системні події. Створювати скрипти, які запускаються до або після операцій Claude Code.

---

## Що ви навчитеся

- Як працюють хуки та коли вони спрацьовують.
- Створення валідації перед комітом (pre-commit).
- Налаштування сповіщень після дій (post-action).
- Написання безпечних скриптів автоматизації.
- Типові патерни використання хуків.

---

## Що таке хуки?

**Хук (hook)** — це скрипт, який запускається автоматично у відповідь на певну подію.

### Події хуків

| Подія | Коли спрацьовує | Типове використання |
|-------|--------|----------|
| **PreToolUse** | Перед запуском інструменту | Валідація запиту |
| **PostToolUse** | Після роботи інструменту | Логування результатів |
| **PreCommit** | Перед комітом у Git | Перевірка якості коду |
| **PostPush** | Після пушу в репозиторій | Сповіщення команди |

---

## Створення першого хука

Хуки — це bash-скрипти, розташовані в `.claude/hooks/`.

### Коди виходу (Exit Codes)
- `exit 0` — **Успіх**: дозволити операцію.
- `exit 1` — **Помилка**: заблокувати операцію та показати помилку.
- `exit 2` — **Попередження**: дозволити, але вивести попередження.

---

## Патерни хуків

### 1. Валідація перед комітом
Блокуйте коміти, які містять секретні ключі або налагоджувальний код:
```bash
#!/bin/bash
# Перевірка на хардкод API ключів
if grep -r "sk_live_" .; then
  echo "❌ ПОМИЛКА: Знайдено Stripe ключ у коді!"
  exit 1
fi
exit 0
```

### 2. Сповіщення після коміту
Логуйте важливі зміни або надсилайте сповіщення:
```bash
#!/bin/bash
COMMIT_MSG=$(git log -1 --pretty=%B)
if echo "$COMMIT_MSG" | grep -i "security"; then
  echo "🔐 Безпековий коміт: $COMMIT_MSG"
fi
exit 0
```

---

## Реєстрація хуків

Хуки реєструються у файлі `.claude/settings.json`:

```json
{
  "hooks": {
    "pre_commit": ["hooks/security-check.sh"],
    "post_commit": ["hooks/notify-team.sh"]
  }
}
```

## Найкращі практики

### ВАРТО РОБИТИ ✅
- Робіть хуки **ідемпотентними** (безпечними для повторного запуску).
- Виводьте зрозумілі повідомлення про помилки.
- Використовуйте `set -e` на початку скрипта.
- Зробіть файл виконуваним: `chmod +x hook.sh`.

### НЕ ВАРТО РОБИТИ ❌
- Не робіть хуки довгими (>5 сек), це дратує розробників.
- Уникайте мережевих запитів у хуках (вони нестабільні).
- Хуки мають лише валідувати, а не змінювати файли без відома користувача.

## Вправа: Створення хука валідації

1. **Створіть скрипт**: `.claude/hooks/pre-commit-check.sh`.
2. **Додайте перевірку** на маркери конфліктів merge (`<<<<<<<`, `=======`).
3. **Налаштуйте тригер**: додайте скрипт у `pre_commit` в налаштуваннях.
4. **Протестуйте**: спробуйте зробити коміт із файлом, що містить маркери конфлікту. Хук має заблокувати дію.
