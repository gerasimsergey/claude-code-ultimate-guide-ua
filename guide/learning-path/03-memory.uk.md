# Модуль 03: Пам'ять та конфігурація

**Час**: 1 година | **Складність**: ⭐⭐ Середній

## Мета

Навчити Claude Code пам'ятати ваші вподобання та правила конкретного проекту. Створити ваш перший файл `CLAUDE.md`.

---

## Що ви навчитеся

- Як працює ієрархія пам'яті Claude Code.
- Створення та структурування файлів `CLAUDE.md`.
- Пріоритетність налаштувань (precedence).
- Проектна vs Глобальна конфігурація.

---

## Ієрархія пам'яті

Claude Code пам'ятає ваші правила на трьох рівнях:

1. **ГЛОБАЛЬНИЙ** (`~/.claude/CLAUDE.md`) — застосовується до ВСІХ ваших проектів (ваш стиль коду, часовий пояс).
2. **ПРОЕКТНИЙ** (`/project/CLAUDE.md`) — тільки для ЦЬОГО проекту (стандарти команди, стек технологій).
3. **ПЕРСОНАЛЬНИЙ** (`/project/.claude/`) — локальні налаштування, які НЕ комітяться в Git (особисті префи, тестові хуки).

**Правило**: Налаштування нижчого рівня перевизначають налаштування вищого.

---

## Створення першого CLAUDE.md

`CLAUDE.md` — це простий markdown-файл у корені проекту, який каже Claude, як працювати.

### Базова структура

```markdown
# Мій Проект

## Технологічний стек
- TypeScript, React 18, Next.js, PostgreSQL

## Стандарти кодування
- Тільки функціональні компоненти.
- Усі експорти мають бути типізовані.
- Максимум 300 рядків на файл.

## Поведінкові правила
- Завжди перевіряй diff перед схваленням.
- Використовуй /plan для будь-яких важливих змін.

## Поточний статус
Зараз працюємо над інтеграцією платіжної системи.
```

---

## Глобальний CLAUDE.md

Для налаштувань, які мають працювати всюди, створіть `~/.claude/CLAUDE.md`:

```markdown
# Мої глобальні вподобання

## Стиль спілкування
- Будь прямим та лаконічним.
- Показуй кроки міркувань.

## Інструменти
- TypeScript для JS проектів.
- Docker для деплою.
- Git для контролю версій.
```

---

## Директорія .claude/

Для локальних налаштувань використовуйте папку `.claude/`. Додайте її в `.gitignore`!

```
my-project/
├── CLAUDE.md           (у Git — правила команди)
├── .claude/
│   ├── settings.json   (НЕ у Git — особисті налаштування)
│   ├── agents/         (кастомні агенти)
│   └── skills/         (кастомні скіли)
```

### settings.json (Приклад)
```json
{
  "model": "claude-3-7-sonnet-latest",
  "context_threshold": 0.7,
  "auto_compact": true,
  "require_diff_review": true
}
```

---

## Найкращі практики

### ВАРТО РОБИТИ ✅
- Оновлюйте `CLAUDE.md` разом із розвитком проекту.
- Додавайте `CLAUDE.md` у Git, щоб допомогти колегам.
- Будьте специфічними у правилах (замість розмитих фраз).

### НЕ ВАРТО РОБИТИ ❌
- Не зберігайте паролі чи секрети в `CLAUDE.md` (використовуйте `.env`).
- Не робіть файл занадто довгим (>500 рядків важко сприймати).
- Уникайте суперечливих правил між глобальним та проектним файлами.

---

## Готовність до наступного модуля

Ви готові, якщо:
✓ Створили файл `CLAUDE.md` у своєму проекті.
✓ Розумієте різницю між глобальними та проектними правилами.
✓ Знаєте, що файли в `.claude/` зазвичай не потрапляють у репозиторій.
✓ Переконалися, що Claude бачить і виконує правила з вашого файлу.
