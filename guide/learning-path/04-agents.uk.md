# Модуль 04: Агенти та спеціалізація

**Час**: 1.5 години | **Складність**: ⭐⭐ Середній

## Мета

Навчитися створювати спеціалізованих агентів для конкретних завдань. Дізнатися, як фокусувати можливості ШІ на точкових проблемах.

---

## Що ви навчитеся

- Що таке агенти і чому вони корисні.
- Створення кастомних агентів за допомогою формату `AGENT.md`.
- Обмеження можливостей агента (ізоляція/пісочниця).
- Коли використовувати агентів замість прямого звернення до Claude.

---

## Що таке агенти?

**Агент** — це спеціалізована версія Claude Code, налаштована для виконання одного конкретного завдання.

### Приклад: Агент-рев'юер коду
Замість того, щоб просити звичайного Claude перевірити код (що вимагає перемикання контексту), ви використовуєте:
```bash
/agent code-reviewer
Перевір цю функцію на наявність багів та проблем із продуктивністю
```

### Звичайний Claude vs Агенти

| Аспект | Звичайний Claude | Агент |
|--------|---------------|-------|
| Сфера | Універсальний | Спеціалізований |
| Контекст | Пам'ятає все | Фокусована пам'ять під задачу |
| Інструменти | Усі доступні | Обмежений набір |
| Швидкість | Багатозадачність | Швидкий в одній справі |

## Створення першого агента

Агенти описуються у файлах `.claude/agents/[name].md`.

### Базова структура
```markdown
---
name: code-reviewer
type: agent
description: Перевіряє код на баги, продуктивність та безпеку
---

# Агент-рев'юер коду

## Мета
Перевіряти код на:
- Логічні помилки
- Вузькі місця продуктивності (O(n²), вкладені цикли)
- Вразливості безпеки
- Відповідність стилю коду

## Інструкції
1. Шукай null pointers та крайові випадки.
2. Перевіряй наявність тестів для змін.
3. Пропонуй покращення ввічливо.
```

---

## Можливості та обмеження

Ви можете обмежувати права агента (sandboxing) за допомогою поля `capabilities`:

```markdown
---
name: junior-dev
capabilities:
  - read_files      # Може читати код
  - run_tests       # Може запускати тести
  - NO: commit      # НЕ може робити коміти
  - NO: push        # НЕ може пушити в Git
---
```
Такий агент може пропонувати правки, але він не зможе випадково зламати основну гілку коду.

## Патерни дизайну агентів

### 1. Контролер якості (Quality Auditor)
Фокусується на метриках: покриття тестами, складність коду, типізація.

### 2. Спеціаліст із безпеки (Security Specialist)
Шукає вразливості OWASP Top 10, витоки секретів та помилки в криптографії.

### 3. Технічний письменник (Doc Writer)
Створює README, API-документацію та опис змін (CHANGELOG).

## Вправа: Створення агента для написання тестів

1. **Створіть файл агента**:
   `.claude/agents/test-writer.md`
2. **Додайте інструкції**:
   Вкажіть патерн Arrange-Act-Assert, вимогу 80% покриття та стиль назв тестів.
3. **Зареєструйте в CLAUDE.md**:
   Додайте опис агента в розділ `## Available Agents`.
4. **Запустіть**:
   `/agent test-writer Напиши тести для src/utils/formatDate.js`

## Коли використовувати агентів

### Використовуйте агентів, якщо: ✅
- Ви робите одну й ту саму задачу регулярно (рев'ю, тести).
- Вам потрібен вузькоспеціалізований інструмент.
- Ви хочете обмежити права ШІ для безпеки.
- Ви будуєте командний воркфлоу.

### Використовуйте звичайного Claude, якщо: ❌
- Ви досліджуєте щось нове або вчитеся.
- Задача нестандартна або одноразова.
- Вам потрібна "жива" розмова на різні теми.
