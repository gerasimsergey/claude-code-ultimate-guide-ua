# Модуль 02: Основний цикл

**Час**: 45 хвилин | **Складність**: ⭐ Початківець

## Мета

Зрозуміти, як насправді працює Claude Code — цикл прийняття рішень, управління контекстом та структурування запитів для максимальної ефективності.

---

## Що ви навчитеся

- Повний цикл взаємодії (запит → аналіз → рішення → дія).
- Як Claude читає та розуміє ваш проект.
- Що таке контекст і чому він критично важливий.
- Режими: Звичайний (Normal), Планування (Plan) та Мислення (Think).
- Як структурувати ефективні запити.

## Як Claude читає ваш проект

Claude не читає все підряд. Він **інтелектуально обирає обсяг (scope)**.

Наприклад, ви просите: "Виправ баг із логіном".
Claude виконає:
1. **Пошук "login"** по всій кодобазі.
2. **Знайде відповідні контролери та сервіси**.
3. **Прочитає ці файли** повністю.
4. **Знайде місця виклику** (хто використовує ці функції?).

Claude **НЕ** читатиме:
- `node_modules` (автоматично ігноруються).
- Історію `.git` (занадто великий обсяг).
- Усі файли проекту без потреби (це повільно і дорого).

## Контекст: Ключова концепція

**Контекст** — це обсяг пам'яті Claude про вашу поточну розмову. Його ліміт становить приблизно **200,000 токенів**.

### Використання команди `/compact`
Коли контекст заповнюється на 70%+, використовуйте:
```bash
/compact
```
Це підсумує історію, видалить старі повідомлення та звільнить приблизно 40% місця, дозволяючи продовжувати роботу в тій же сесії.

## Режими: Normal vs Plan vs Think

### Звичайний режим (Normal — за замовчуванням)
Claude пропонує зміни одразу після аналізу. Найкраще для швидких правок та рефакторингу невеликих ділянок.

### Режим планування (`/plan`)
Claude спочатку пропонує покроковий план, чекає на ваше схвалення, і тільки потім переходить до написання коду. Використовуйте для складних фіч та ризикованих змін.

### Режим мислення (`/think`)
Claude показує розширений ланцюжок міркувань (reasoning). Корисно для архітектурних рішень та аналізу продуктивності.

---

## Структурування ефективних запитів

Використовуйте фреймворк **ШО (WHAT), ДЕ (WHERE), ЯК (HOW), ПЕРЕВІРКА (VERIFY)**:

| Частина | Суть | Приклад |
|------|---------|---------|
| **ШО** | Мета | "Виправ баг із нульовим покажчиком" |
| **ДЕ** | Обсяг | "у `src/auth/login.js` у рядку 45" |
| **ЯК** | Обмеження | "не змінюючи сигнатуру API" |
| **ПЕРЕВІРКА** | Результат | "Усі існуючі тести мають пройти" |

---

## Практична вправа: Повний цикл

1. **Задайте запит за фреймворком:**
   "Створи утиліту для валідації email. ДЕ: у `src/utils/validators.js`. ЯК: експортуй як `validateEmail(email)`, повертай boolean. ПЕРЕВІРКА: напиши тести у `tests/validators.test.js`."
2. **Claude проаналізує проект** та запропонує рішення.
3. **Уважно перегляньте diff**. Перевірте логіку регулярного виразу.
4. **Схваліть зміни**.
5. **Готово!** У вас є протестована функція.
