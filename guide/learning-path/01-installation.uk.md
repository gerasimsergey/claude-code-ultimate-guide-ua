# Модуль 01: Встановлення та налаштування

**Час**: 15 хвилин | **Складність**: ⭐ Початківець

## Мета

Встановити Claude Code у вашій системі та переконатися в його працездатності, виконавши першу команду.

---

## Що ви навчитеся

- Встановлювати Claude Code для вашої платформи (macOS / Linux / Windows).
- Розуміти базовий цикл "запит → відповідь".
- Запускати свою першу команду.
- Користуватися системою допомоги.

---

## Встановлення

### macOS (Рекомендовано)

```bash
brew install anthropic/tap/claude-code
```

Перевірка:
```bash
claude --version
```

### Linux

```bash
curl -sSL https://dl.claudecode.com/install.sh | bash
```

Перевірка:
```bash
claude --version
```

### Windows

Завантажте інсталятор з https://dl.claudecode.com/windows або використайте PowerShell:

```powershell
iex ((New-Object System.Net.WebClient).DownloadString('https://dl.claudecode.com/install.ps1'))
```

### Docker (Будь-яка платформа)

```bash
docker run -it anthropic/claude-code:latest
```

## Перший запуск

Перейдіть у директорію будь-якого проекту та запустіть Claude:

```bash
cd ~/my-project
claude
```

Ви побачите щось на зразок:

```
Claude Code v2.x.x ready
Project: ~/my-project (git: main)
Context: 0% · Tokens available: 200,000

Type /help for commands or ask me anything
>
```

---

## Основні команди

| Команда | Призначення |
|---------|---------|
| `/help` | Показати всі доступні команди |
| `/status` | Перевірити використання контексту та стан сесії |
| `/clear` | Почати заново (очищає історію розмов) |
| `Ctrl+C` | Скасувати поточну операцію |
| `/exit` | Закрити Claude Code |

## Ваші перші 5 хвилин

### Вправа 1: Перегляд доступних команд
```bash
/help
```

Зверніть увагу на категорії:
- **Workflow (Воркфлоу)**: `/plan`, `/rewind`, `/think`.
- **Navigation (Навігація)**: `/goto`, `/read`.
- **Memory (Пам'ять)**: автоматичне завантаження знань при старті.

### Вправа 2: Перевірка стану сесії
```bash
/status
```

Ви побачите:
- Відсоток використання контексту.
- Доступні токени.
- Поточний проект та гілку Git.

### Вправа 3: Задайте питання
Введіть: `Які файли є в моєму проекті?`

Claude проаналізує структуру та відповість. Це і є основний цикл:
`Ваш запит → Claude читає файли → Claude пропонує зміни → Ви перевіряєте → Застосовуєте`

### Вправа 4: Перевірка запропонованих змін
Якщо Claude запропонує зміни в коді, ви побачите:
1. Опис змін.
2. Вікно `diff` (що додається/видаляється).
3. Питання про схвалення або відхилення.

**Правило**: Завжди перевіряйте diff-и перед схваленням. Це захистить вас від неочікуваних змін.

## Ключові концепції

### Сесії
Кожен запуск `claude` — це нова **сесія**. Це розмова, яка триває, поки ви не вийдете з Claude Code. Сесії прив'язані до одного проекту.

### Контекст
Це обсяг пам'яті Claude про поточну розмову (0-100%).
- 0-50%: Працюйте вільно.
- 70%+: Рекомендується команда `/compact` для очищення місця.
- 90%+: Система змусить вас очистити історію.

---

## Вирішення проблем

### "claude: command not found"
Встановлення не завершилося. Спробуйте встановити ще раз за інструкцією вище.

### "Project not found"
Переконайтеся, що ви знаходитесь у директорії з файлом `package.json`, `.git` або іншим маркером проекту.
