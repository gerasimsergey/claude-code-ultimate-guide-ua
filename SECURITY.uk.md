# Політика безпеки (Security Policy)

## Сфера дії (Scope)

Цей репозиторій містить **документацію та навчальні шаблони** для Claude Code. Він не містить виконуваного коду, який обробляє вхідні дані користувача або працює в продакшен-середовищах.

**Питання безпеки, специфічні для цього репозиторію:**
- Точність документації щодо практик безпеки.
- Якість коду шаблонів та патерни безпеки.
- Цілісність бази даних загроз ([`machine-readable/threat-db.yaml`](./machine-readable/threat-db.yaml)).

**Поза сферою дії:**
- Вразливості безпеки в самому CLI Claude Code → повідомляйте [Anthropic](https://github.com/anthropics/claude-code/security).
- Питання безпеки в MCP серверах → повідомляйте відповідним мейнтейнерам серверів.

## Повідомлення про проблеми безпеки

Якщо ви виявили проблему безпеки, пов'язану з цим посібником (наприклад: шкідливий шаблон, неправильна порада з безпеки, неточності в базі загроз), будь ласка:

1. **Email**: florian.bruniaux@methode-aristote.fr
   - Тема: `[SECURITY] Claude Code Guide - Brief Description`
   - Додайте: Уражений файл/розділ, опис, оцінку впливу.

2. **GitHub Private Disclosure**: Використовуйте [Security Advisories](../../security/advisories/new) для чутливих питань.

**SLA відповіді**: Ми прагнемо відповісти протягом 48 годин і випустити виправлення протягом 7 днів для критичних проблем.

## Ресурси з безпеки

Цей посібник підтримує розгалужену документацію з безпеки:

- **[Security Hardening Guide](./guide/security-hardening.md)** — перевірка MCP, захист від ін'єкцій, воркфлоу аудиту.
- **[Threat Database](./machine-readable/threat-db.yaml)** — 18 CVE, 341 шкідливий скіл.
- **[Security Hooks](./examples/hooks/)** — 30 продакшен хуків (bash + PowerShell).
- **[Security Commands](./examples/commands/)** — `/security-check`, `/security-audit`, `/update-threat-db`.

## Підтримка безпеки

**Оновлення бази даних загроз**: База даних розвідки загроз оновлюється на основі:
- Оголошень CVE та повідомлень про безпеку.
- Звітів спільноти про шкідливі скіли/MCP сервери.
- Бюлетенів безпеки Anthropic.
- Академічних досліджень (наприклад, статті про prompt injection).

**Графік аудиту**:
- Щотижневий огляд нових MCP серверів та скілів.
- Щомісячний аудит точності документації з безпеки.
- Щоквартальне повне оновлення бази даних загроз.

**Останнє оновлення**: 2026-02-11 (v3.26.0)

## Скоординоване розкриття інформації (Coordinated Disclosure)

Якщо ви є дослідником безпеки та знайшли проблеми, що впливають на декілька репозиторіїв в екосистемі Claude Code:

1. Спочатку напишіть нам (перевага надається скоординованому розкриттю).
2. Ми скоординуємо дії з іншими мейнтейнерами за потреби.
3. Терміни публічного розкриття: 90 днів або після виправлення, залежно від того, що настане раніше.

## Подяки

Ми дякуємо дослідникам безпеки, які зробили свій внесок у покращення безпекового контенту цього посібника через відповідальне розкриття інформації.
