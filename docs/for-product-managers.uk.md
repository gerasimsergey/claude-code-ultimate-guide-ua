# Claude Code — Для продукт-менеджерів та дизайнерів

> Вам не потрібно ставати розробником, щоб отримати користь від Claude Code. Розуміння того, як він працює — і що він може зробити безпосередньо для вас — змінить вашу взаємодію з командою та те, що ви зможете створювати самостійно.

---

## Два різні сценарії використання

**1. Ви працюєте разом із розробниками, які використовують Claude Code**
Розуміння інструменту допоможе вам писати кращі специфікації, швидше знімати блокування та вести більш предметні розмови про витрати ресурсів та компроміси.

**2. Ви хочете використовувати його самостійно**
"Vibe coding" — це реальність. Продакт-менеджери все частіше створюють прототипи функцій, пишуть скрипти та генерують аналіз даних без допомоги розробника. Claude Code робить це доступним, якщо ви готові вивчити основи.

---

## Якщо ви працюєте з розробниками, які використовують Claude Code

### Що змінюється у вашому робочому процесі

**Специфікації стають контекстом** — ваші PRD, критерії прийняття та крайові випадки можуть жити у файлі `CLAUDE.md` у корені гілки з фічею. Claude Code розробника читає їх автоматично. Кращі специфікації = менше уточнень.

**Ви можете перевіряти роботу, створену за допомогою ШІ** — коли розробник каже: "Claude написав більшу частину цього", ви можете попросити Claude Code пояснити, що було згенеровано, чому і які є компроміси. Знання коду не обов'язкове.

**Змінюються очікування щодо швидкості** — створення шаблонного коду, тестів, документації, міграцій: все це займає в рази менше часу з Claude Code. Якщо ваші оцінки спринтів досі відображають зусилля до появи ШІ, вони помилкові.

## Якщо ви хочете використовувати Claude Code самостійно

### Що ви можете зробити без досвіду програмування

- **Створити прототип функції** — опишіть те, що хочете, і Claude Code згенерує робочу демо-версію.
- **Обробити дані** — пишіть SQL або Python скрипти, описуючи свої потреби звичайною мовою.
- **Автоматизувати рутинні завдання** — наприклад, перетворення Excel у CSV та створення форматованого звіту без допомоги розробника.
- **Згенерувати тест-кейси** — вставте критерії прийняття і отримайте набір тестів.
- **Зрозуміти існуючий код** — питання "Що робить ця функція?" працює краще, ніж Stack Overflow для вашої кодобази.

### Що насправді потрібно для старту

1. Встановити Claude Code (5 хв) → [Розділ 1.1 Посібника](../guide/ultimate-guide.md#11-installation).
2. Вивчити одну формулу промпту → [Розділ 2.8 — Структуровані промпти](../guide/ultimate-guide.md#28-structured-prompting-with-xml-tags).
3. Зрозуміти, як давати контекст → [Розділ 3.1 — CLAUDE.md](../guide/ultimate-guide.md#31-memory-files-claudemd).

Цього достатньо для першого тижня.
