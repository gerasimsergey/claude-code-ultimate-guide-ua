## Процес Pull Request (PR)

### 1. Fork & Clone

```bash
git clone https://github.com/YOUR_USERNAME/claude-code-ultimate-guide.git
cd claude-code-ultimate-guide
```

### 2. Створіть гілку (Branch)

```bash
git checkout -b fix/typo-in-section-3
# або
git checkout -b feature/add-debugging-guide
```

### 2.5. Встановіть Pre-commit Hooks

Цей репозиторій використовує pre-commit хуки для контролю якості (markdown lint, YAML валідація, узгодженість версій тощо). Налаштуйте їх один раз:

```bash
# macOS/Linux
pip install pre-commit
pre-commit install

# Windows (PowerShell)
pip install pre-commit
pre-commit install
```

Pre-commit тепер запускатиметься автоматично при `git commit`. Ви також можете запустити його вручну:

```bash
pre-commit run --all-files
```

**Що перевіряють хуки:**
- ✓ Узгодженість форматування Markdown.
- ✓ Валідність синтаксису YAML.
- ✓ Узгодженість файлу VERSION по всьому репозиторію.
- ✓ Відсутність битих симлінків або конфіденційних файлів.

### 3. Внесіть зміни

Дотримуйтесь [Content Guidelines](#content-guidelines) нижче.

### 4. Протестуйте ваші зміни

- Перевірте рендеринг markdown.
- Протестуйте фрагменти коду на вашій платформі.
- Переконайтеся, що всі посилання працюють.

### 5. Надішліть PR

Вкажіть:
- Чіткий опис змін.
- Чому ці зміни потрібні.
- Що саме ви протестували.

## Content Guidelines (Рекомендації щодо контенту)

### Стиль написання

- **Стислість**: Буліти (bullet points) > довгі абзаци.
- **Практичність**: Додавайте приклади для кожної концепції.
- **Кросплатформеність**: Підтримка macOS/Linux ТА Windows.
- **Точність**: Тестуйте весь код перед відправкою.

### Структура документації

```markdown
## Назва розділу

Короткий вступ (1-2 речення).

### Підрозділ

| Колонка 1 | Колонка 2 |
|----------|----------|
| Дані     | Дані     |

**Приклад:**
```bash
приклад коду тут
```
```

### Код для конкретних платформ

Завжди надавайте обидва варіанти, якщо команди відрізняються:

```bash
# macOS/Linux
~/.claude/settings.json

# Windows
%USERPROFILE%\.claude\settings.json
```

## Чек-лист якості

Перед відправкою:

- [ ] Markdown рендериться правильно (попередній перегляд на GitHub).
- [ ] Усі посилання працюють.
- [ ] Фрагменти коду протестовані на вашій платформі.
- [ ] Надано еквіваленти для Windows (якщо застосовно).
- [ ] Перевірено орфографію.
- [ ] Дотримано існуючого стилю.

## Внески для Windows (Особливо вітаються!)

Мейнтейнер працює на macOS. Якщо ви користувач Windows:

- Тестуйте PowerShell скрипти на версії PS 5.1+.
- Перевіряйте обробку шляхів.
- Повідомляйте про специфічні для Windows проблеми.
- Додавайте альтернативи у вигляді batch-файлів, де це можливо.

**Ваш внесок особливо цінний!**
