# Ідеї для дослідження (Ideas to Dig)

> Теми для майбутніх покращень посібника. Відібрані та перевірені.

## Виконано

### Зміцнення безпеки MCP ✅
Об'єднане дослідження безпеки, що охоплює вразливості MCP, промпт-ін'єкції та виявлення секретів.

**Завершено**: [Посібник зі зміцнення безпеки](./guide/security/security-hardening.md) охоплює:
- CVE-2025-53109/53110, 54135, 54136 із заходами пом'якшення.
- Робочий процес перевірки MCP з 5-хвилинним чек-лістом аудиту.
- "Білий список" MCP (перевірений спільнотою).
- Техніки обходу промпт-ін'єкцій (Unicode, ANSI, нульові байти).
- Порівняння інструментів виявлення секретів (Gitleaks, TruffleHog, GitGuardian).
- Процедури реагування на інциденти.
- 3 нових хуки: `unicode-injection-scanner.sh`, `repo-integrity-scanner.sh`, `mcp-config-integrity.sh`.

## Високий пріоритет

### Команди, натхненні gstack (відкладено після аудиту 26.03.2026)

Аудит [gstack](https://github.com/garrytan/gstack) (набір інструментів Гаррі Тана для Claude Code) виявив прогалини. Пріоритетні елементи вже впроваджено (`/investigate`, `/qa`, `/canary`, `/land-and-deploy`, оновлення `/review-pr`). Решта елементів відкладена сюди.

**`/autoplan`** — єдина команда, яка автоматично об'єднує CEO → Design → Eng review із закодованими принципами прийняття рішень. Поділ рішень на механічні (авто-рішення) та рішення на основі "смаку" (виносяться на користувача). Джерело: `gstack/autoplan/SKILL.md`.

**`/office-hours`** — діагностика продукту в стилі YC з 6 ключовими питаннями перед будь-яким плануванням. Передує `/plan-ceo-review`. Записує дизайн-бриф у `~/.claude/projects/`. Джерело: `gstack/office-hours/SKILL.md`.

**`/design-review`** — візуальний аудит сайту + цикл виправлень. Робить скріншот URL, аналізує відступи/типографіку/колір/контраст, виявляє патерни "AI-slop" (фіолетові градієнти, 3-колонкові сітки, вирівнювання всього по центру). Джерело: `gstack/design-review/SKILL.md`.

**`/benchmark`** — виявлення регресій Core Web Vitals (TTFB, FCP, LCP, завантаження DOM, розміри бандлів). Базове порівняння з порогами регресії. Джерело: `gstack/benchmark/SKILL.md`.

**`/retro`** — щотижневе ретро з аналізом git-логів по кожному учаснику. Джерело: `gstack/retro/SKILL.md`.

### Каталог серверів MCP
Вичерпний список серверів MCP з реальними сценаріями використання.
**Теми:**
- Доступні сервери за категоріями (інструменти розробника, бази даних, API).
- Бенчмарки продуктивності порівняно з нативними інструментами.
- Рівні довіри до безпеки кожного сервера.

## Відкладені ідеї (Discarded Ideas)

| Ідея | Причина відхилення |
|------|------------------|
| Посібник з fine-tuning LLM | Поза межами — користувачі не керують навчанням моделей |
| Внутрішня архітектура моделей | Занадто теоретично, не практично |
| Оптимізація ціни за токени | Змінюється надто часто, використовуйте офіційні доки |
| Протокол A2A (Agent-to-Agent) | Claude Code — це одноагентна система з субагентами |
| Canary/Blue-Green деплої | Патерни інфраструктури, не релевантно для CLI |
| Порівняння з іншими інструментами | Швидке застарівання |
